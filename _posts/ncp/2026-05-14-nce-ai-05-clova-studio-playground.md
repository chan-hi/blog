## 핵심 요약

- CLOVA Studio는 HyperCLOVA X 기반 생성형 AI 개발 플랫폼입니다.
- 주요 메뉴는 플레이그라운드, 튜닝, 익스플로러, 스킬 트레이너, 포럼, 내 작업 등입니다.
- 플레이그라운드는 코딩 지식 없이 프롬프트와 파라미터 설정만으로 모델 응답을 테스트하고 API로 서비스에 적용할 수 있는 공간입니다.
- 시험에서는 모델별 특징, 파라미터 의미, 활용 작업 유형을 구분하는 것이 중요합니다.

## 플레이그라운드

플레이그라운드는 AI 지식이나 코딩 지식이 없어도 사용할 수 있는 테스트 공간입니다. 프롬프트를 입력하고 파라미터를 설정하여 텍스트를 생성합니다. 원문에서는 플레이그라운드에 4,096 토큰 제한이 있다고 설명합니다.

주요 구성 요소는 다음과 같습니다.

| 구성 요소 | 설명 |
|---|---|
| 모델 | 사용할 모델 선택 |
| 파라미터 | 텍스트 생성을 위한 값 설정 |
| 실행 | 작업 실행 |
| 시스템 | 모델의 역할, 규칙, 말투 등을 지정하는 프롬프트 영역 |
| 사용자 | 사용자의 질문 또는 입력 영역 |
| 결과 | 어시스턴트 응답 출력 영역 |

파라미터 목록은 다음과 같습니다.

- Engine
- Top K
- Top P
- Maximum tokens
- Temperature
- Repetition penalty
- Stop sequences
- Seed

## 제공 엔진과 모델 선택

CLOVA Studio는 HyperCLOVA X 계열 모델을 제공합니다.

| 모델 | 핵심 특징 |
|---|---|
| HCX-007 | 심층적 사고 과정을 반영한 하이브리드 추론 모델. 수학, 과학, 언어 추론, 논술 등 복잡한 문제 해결에 활용 |
| HCX-005 | 텍스트, 이미지, 표 등 다양한 형태의 데이터를 처리하고 이해. 도메인 특화 지식과 추론에 적합 |
| HCX-DASH-002 | 32K context length를 지원하는 경량화 텍스트 특화 모델. 내부 지식 검색처럼 실시간 서비스에 강점 |
| HCX-003 | 기존 HyperCLOVA X 기본 모델 계열 |
| HCX-DASH-001 | 경량 모델. 빠른 속도와 단순 작업에 유용 |

사용자 데이터를 학습한 튜닝 모델도 선택할 수 있습니다.

## 모델 히스토리와 로드맵

원문은 2021년 HyperCLOVA, 2023년 HyperCLOVA X, 2024~2025년 모델 업데이트와 기능 확장을 제시합니다.

주요 흐름은 다음과 같습니다.

- HyperCLOVA
- HyperCLOVA X
- Foundation Model 업데이트
- 국내 최초 Vision Model
- 국내 최대 Context Size, 128K
- 추론 모델
- 플래그십 모델 HCX-005, HCX-007
- 경량 모델 HCX-DASH-001, HCX-DASH-002
- 오픈소스 HCX-SEED, HCX-THINK
- Skill Trainer, Advanced Tuning, Explore APIs, LangChain compatibility, Embedding V2, Router, Function Call, Structured Output, RAG Reasoning API, Re-ranker API

## HCX-DASH-002

HCX-DASH-001과 HCX-DASH-002는 경량 엔진입니다. 빠른 속도와 처리 능력이 장점이며 비교적 단순한 작업에 유용합니다.

원문은 HCX-DASH-002가 HCX-DASH-001보다 KMMLU, MMLU 등에서 개선된 성능을 보인다고 설명합니다.

## HCX-007 추론 모델

HCX-007은 복잡한 문제 해결에 강한 추론 모델입니다.

- 수학/과학 문제 해결
- 언어 추론
- 논술
- 추론 모드와 비추론 모드 선택 가능
- Thinking Effort 기능으로 사고 깊이를 단계적으로 조정

시험에서는 `HCX-007 = 복잡한 문제와 추론에 강한 모델`로 기억하면 됩니다.

## HCX-005 멀티모달 모델

HCX-005는 텍스트뿐 아니라 이미지와 표 등 다양한 데이터를 처리할 수 있습니다.

강점은 다음과 같습니다.

- 한국어 문서 및 이미지 내 텍스트 이해
- 문화와 유머, 밈 이해
- Entity Recognition: 인명, 장소, 제품, 음식 등 의미 단위 인식
- K-GED처럼 이미지 형태의 사지선다 문제 이해 및 답변 평가

## 업무 목적별 모델 선택

| 항목 | HCX-005 | HCX-DASH-002 |
|---|---|---|
| 강점 | 도메인 특화 지식과 추론이 필요한 Use Case | 일부 도메인 특화 + RAG 기반 실시간 서비스 |
| 추론 능력 | 준수함 | 부족함, 복잡한 업무 수행에는 부적합 |
| 문장 생성 | 상황에 맞는 문장 생성 | 문장 생성은 가능하나 HCX-005 대비 열위 |
| 처리 속도 | HCX-DASH-002 대비 느림, 프롬프트 추론 시간 필요 | 실시간 서비스 가능 |
| 튜닝 | Use Case에 따라 일부 튜닝 필요 | 상용화를 위해 튜닝 필요 |
| 예시 | 도메인 특화 문서 초안, 도메인 특화 Assistant | 내부 지식 검색, 매뉴얼/업무 가이드 챗봇 |

## 파라미터 정리

### Top P

Top P는 누적 확률 기준입니다. 설정한 누적 확률에 포함되지 않는 토큰은 후보에서 제외합니다.

- 범위: 0~1
- 예: Top P = 0.8이면 상위 누적 확률 80%에 포함된 토큰에서만 생성
- 권장: 특수한 경우가 아니라면 0.8~1

### Top K

Top K는 확률값이 높은 K개의 토큰 중 하나를 선택하게 하는 기준입니다.

- 범위: 0~128
- 예: Top K = 5이면 확률이 높은 5개 토큰 중 하나 선택
- 권장: 일반적으로 0

### Maximum Tokens

Maximum tokens는 AI가 생성하는 결과값의 최대 길이입니다. 값이 높을수록 긴 결과를 출력합니다.

토큰은 문장을 가장 작은 단위의 단어 조각으로 나눈 것입니다. 한국어 단어는 보통 형태소 단위의 1~2토큰으로 나뉩니다. 토큰화는 언어 모델이 자연어를 처리하기 위한 필수 과정입니다.

| 모델 | 입력 가능 유형 | Context length | 최대 출력 토큰 | 튜닝 지원 | Function calling | Structured Outputs | 추론 지원 |
|---|---|---:|---:|---|---|---|---|
| HCX-007 | 텍스트 | 128,000 | 32,768 | 지원 | 지원 | 미지원 | 하이브리드 |
| HCX-005 | 텍스트/이미지 | 128,000 | 4,096 | 지원 | 지원 | 미지원 | 미지원 |
| HCX-DASH-002 | 텍스트 | 32,000 | 4,096 | 지원 | 미지원 | 지원 | 미지원 |
| HCX-003 | 텍스트 | 8,192 | 7,600 | 미지원 | 미지원 | 미지원 | 미지원 |
| HCX-DASH-001 | 텍스트 | 4,096 | 3,500 | 지원 | 미지원 | 미지원 | 미지원 |

### Temperature

Temperature는 확률 분포에 가중치 변화를 주어 문장의 다양성을 조절합니다.

| 값 | 특징 | 적합한 작업 |
|---|---|---|
| 낮음 | 정형적이고 예측 가능한 결과 | 분류, 요약, 사실 기반 답변 |
| 높음 | 다양한 문장 생성, 품질 편차 가능 | 소설, 마케팅 문구, 창의적 글쓰기 |

원문은 Top P를 고정한 상태에서 필요에 따라 Temperature를 조절하는 것을 권장합니다.

### Repetition penalty

Repetition penalty는 반복되는 토큰에 감점 요소를 부여하여 동일한 결과 반복을 줄이는 값입니다.

- 범위: 0~10
- 값이 낮으면 같은 결과 반복 가능성이 높아짐
- 값이 높으면 같은 결과 반복 가능성이 낮아지고 새로운 output 생성 가능성이 높아짐

### Stop sequences

Stop sequences는 결과 생성을 중단할 때 사용할 문자열입니다.

- 여러 개 등록 가능
- 결과에 Stop sequence 중 하나가 포함되면 그 이전 내용까지만 출력
- 예: `원숭이 엉덩이는 빨개` 입력 후 Stop sequence를 `사과`로 지정하면 `사과` 이전까지만 출력

### Seed

Seed는 결과물의 일관성을 조정하는 값입니다.

- 같은 Seed를 사용하면 확률 기반 모델을 여러 번 실행해도 같은 결과를 얻을 가능성이 높음
- Seed = 0이면 무작위 출력
- 직접 입력 범위: 1~4,294,967,295
- 단, 결과의 완전성은 보장하지 않으며 미세한 조건 변화로 결과가 달라질 수 있음

### HCX-005 비전 모델 업무 자동화

원문은 HCX-005를 활용한 다음 예시를 제시합니다.

- 인보이스 정보 추출: 공급자 업체명, 사업자번호, 날짜, 총액 등 추출
- 회의 자료 정리: 회의 목적, 핵심 키워드, 주요 논의 주제, 도식/흐름, 결정사항 정리
- 화이트보드 정보 추출: 손글씨, 도식, 화살표, 표를 사람이 이해하듯 해석
- 발표대본 자동 생성
- 외국어 텍스트 추출/번역/요약
- 상품 이미지 기반 마케팅 문구 생성

## 시험 암기 포인트

- Top P는 누적 확률, Top K는 후보 개수입니다.
- Temperature는 다양성, Repetition penalty는 반복 억제, Stop sequences는 출력 중단 문자열, Seed는 결과 일관성입니다.
- HCX-005는 멀티모달과 도메인 추론, HCX-DASH-002는 경량/실시간/RAG 기반 서비스에 적합합니다.
- 플레이그라운드는 프롬프트와 파라미터로 빠르게 테스트하고 API 적용까지 이어지는 기능입니다.

## 이야기: 클라우드빵집 대박 사건

어느 날 클라우드빵집이 인스타그램 핫플에 소개됐다.

> "하루 주문이 100배 폭발! 서버가 버티질 못해!"

기존 가상 서버(VM)의 성능 한계를 체감한 민준.  
NCP 컨설턴트가 두 가지 옵션을 제안했다.

> "**Bare Metal Server**는 하이퍼바이저를 없애고 물리 서버를 직접 쓰는 거예요.  
> **GPU Server**는 AI 연산에 특화된 GPU를 붙인 서버고요."

## 🔑 핵심 개념 1: Bare Metal Server (베어메탈)

**Bare Metal** = 하이퍼바이저 없이 **물리 서버에 직접 OS를 설치**하는 방식.  
다른 고객과 물리 서버를 공유하지 않는다. **나만의 전용 서버**.

### Bare Metal의 장점
- 하이퍼바이저 오버헤드 없음 → **최고 수준의 성능**
- 다른 VM의 영향을 받지 않음 (Noisy Neighbor 문제 없음)
- 클라우드의 편리함 + 물리 서버의 안정성

### Bare Metal의 제약사항 (⭐ 시험 단골!)

| 항목 | 가능 여부 |
|------|----------|
| 스토리지 추가 | **불가** ❌ |
| 스토리지 크기 변경 | **불가** ❌ |
| 오토스케일링 | **불가** ❌ |
| RAID 구성 선택 | **가능** ✅ |

> ⚠️ **Bare Metal 3불가**: 스토리지 추가 불가, 크기 변경 불가, 오토스케일링 불가!

---

### Bare Metal RAID 구성 선택 (⭐ 시험 단골!)

Bare Metal은 **용도에 따라 RAID 방식을 선택**할 수 있다.

| RAID 종류 | 방식 | 특징 | **선택 용도** |
|-----------|------|------|-------------|
| **RAID 1+0** | 미러링(복제) + 스트라이핑(분산) | 성능 + 안정성 | **DBMS 등 고성능/고안정성** |
| **RAID 5** | 패리티 분산 저장 | 용량 효율적 | **일반 웹 서비스** |

```
[RAID 선택 기준]
DB 서버 (MySQL, Oracle 등)  → RAID 1+0
웹 서버, 파일 서버 등        → RAID 5
```

> 💡 **RAID 1+0** = "미러링(1) + 스트라이핑(0)"의 결합  
> 디스크 절반은 복제본 → 비용은 높지만 속도와 안정성 모두 확보

## 🔑 핵심 개념 2: GPU Server

클라우드빵집이 AI 추천 시스템을 도입하기로 했다.  
"어떤 빵을 좋아하는 고객인지 딥러닝으로 예측해보자!"

GPU 서버는 **병렬 연산**에 특화된 GPU를 장착한 서버.  
이미지 처리, 딥러닝 학습/추론, 과학 연산에 사용.

### GPU 서버의 핵심 특징 (⭐ 시험 단골!)

| 항목 | 내용 |
|------|------|
| **GPU 할당 방식** | **Pass Through** 방식 ← 이게 핵심! |
| **GPU 할당 단위** | **장(개) 단위** (Core 단위가 아님) |

> ⚠️ **시험 포인트**  
> NCP GPU 서버는 **NVIDIA GRID 방식이 아닌 Pass Through 방식**!  
> - **NVIDIA GRID**: GPU를 쪼개서 여러 VM이 공유 (vGPU)  
> - **Pass Through**: GPU 장 단위로 VM에 **전용 할당** → 성능 최대

### GPU 서버 모델별 스펙

| 모델 | 서버 타입 | GPU 최대 수 | 특이사항 |
|------|----------|------------|---------|
| **T4** | VM | 최대 **2장** | - |
| **V100** | VM | 최대 **4장** | - |
| **A100** | **Bare Metal** | 최대 **8장** | 로컬 NVMe 디스크 기본 제공 |

### GPU 서버의 제약사항

| 항목 | 가능 여부 |
|------|----------|
| 오토스케일링 | **불가** ❌ |
| 네트워크 인터페이스(NIC) 추가 | **이용 불가** ❌ |

---

## 전체 비교 요약

| 항목 | 일반 VM | Bare Metal | GPU Server |
|------|---------|------------|-----------|
| 하이퍼바이저 | 있음 | **없음** | 있음 (Pass Through) |
| 스토리지 추가 | 가능 | **불가** | 가능 |
| 오토스케일링 | 가능 | **불가** | **불가** |
| 주요 용도 | 일반 서비스 | 고성능 DB/전용 서버 | AI/딥러닝/이미지처리 |

## 📝 시험 대비 체크리스트

- [ ] Bare Metal: 하이퍼바이저 **없음**, 물리 서버 직접 사용
- [ ] Bare Metal: 스토리지 추가 **불가**, 크기 변경 **불가**, 오토스케일링 **불가**
- [ ] Bare Metal RAID: **RAID 1+0** → DB/안정성, **RAID 5** → 일반 웹
- [ ] GPU: **Pass Through 방식** (NVIDIA GRID 아님!)
- [ ] GPU 할당 단위: **장(개) 단위** (Core 단위 아님)
- [ ] GPU VM: T4 최대 **2장**, V100 최대 **4장**
- [ ] GPU Bare Metal: A100 최대 **8장**, NVMe 기본 제공
- [ ] GPU 서버: 오토스케일링 **불가**, NIC **이용 불가**

## 🔑 핵심 개념 1: Bare Metal Server (베어메탈)

**Bare Metal** = 하이퍼바이저 없이 **물리 서버에 직접 OS를 설치**하는 방식.  
다른 고객과 물리 서버를 공유하지 않는다. **나만의 전용 단독 서버**.

단독으로 사용할 수 있는 고성능 물리 서버를 클라우드 형태로 제공하기 때문에, 성능에 민감한 서비스도 안정적으로 운영할 수 있다.

### Bare Metal의 장점
- 하이퍼바이저 오버헤드 없음 → **최고 수준의 성능**
- 다른 VM의 영향을 받지 않음 (Noisy Neighbor 문제 없음)
- 클라우드의 편리함 + 물리 서버의 안정성
- **적합한 RAID 구성 방식을 선택**할 수 있음 (RAID 1+0 / RAID 5)

### Bare Metal 지원 OS

CentOS, ORACLE Linux, Red Hat Enterprise Linux, Ubuntu Server, Windows를 제공한다.

### Bare Metal 제공 사양 (⭐ 시험 단골!)

| 구성 | CPU | 코어 수 | 메모리 | 디스크 |
|------|-----|--------|-------|-------|
| Single | Intel Xeon Silver 4215 (2.6GHz) | 8개 | 128GB | 4 × 1.9TB SSD |
| Dual | Intel Xeon Silver 4214 (2.2GHz) | 24개 | 256GB | 4 × 1.9TB SSD |
| Dual | Intel Xeon Gold 5220 (2.2GHz) | 36개 | 384GB | 8 × 1.9TB SSD |
| Dual | Intel Xeon Gold 6248R (3.0GHz) | 48개 | 512GB | 8 × 1.9TB SSD + 2 × 240GB SSD |
| **GPU (A100)** | **Dual AMD Epyc Milan 7313 (3.0GHz)** | **32개** | **2TB** | **Tesla A100 GPU 8EA + 4 × 7.68TB NVMe SSD** |

> 💡 **NVMe(Non-Volatile Memory express)**: PCIe 인터페이스 기반 고속 저장장치 규격.  
> SATA 대비 최대 6배 이상의 속도를 낼 수 있어, SSD 성능을 최대한 활용할 수 있도록 개발된 초고속 데이터 전송 규격이다.

### Bare Metal의 제약사항 (⭐ 시험 단골!)

| 항목 | 가능 여부 |
|------|----------|
| 스토리지 추가 | **불가** ❌ |
| 스토리지 크기 변경 | **불가** ❌ |
| 오토스케일링 | **불가** ❌ |
| SSD 스토리지 이용 | 가능 ✅ |
| 네트워크 인터페이스 이용 | 가능 ✅ |
| RAID 구성 선택 | **가능** ✅ |

> ⚠️ **Bare Metal 3불가**: 스토리지 추가 불가, 크기 변경 불가, 오토스케일링 불가!

### IPMI (Intelligent Platform Management Interface)

Bare Metal 서버에는 **IPMI**가 제공된다. 하드웨어를 제어하고 모니터링하기 위한 별도의 장치로, 원격에서 하드웨어의 전원을 제어하고 하드웨어를 모니터링하며 펌웨어를 제어할 수 있다.

---

### Bare Metal RAID 구성 선택 (⭐ 시험 단골!)

Bare Metal은 **용도에 따라 RAID 방식을 선택**할 수 있다.

| RAID 종류 | 방식 | 특징 | **선택 용도** |
|-----------|------|------|-------------|
| **RAID 1+0** | 미러링(복제) + 스트라이핑(분산) | 성능 + 안정성 | **DBMS 등 고성능/고안정성** |
| **RAID 5** | 패리티 분산 저장 | 용량 효율적 | **일반 웹 서비스** |

```
[RAID 선택 기준]
DB 서버 (MySQL, Oracle 등)  → RAID 1+0
웹 서버, 파일 서버 등        → RAID 5
```

> 💡 **RAID 1+0** = "미러링(1) + 스트라이핑(0)"의 결합  
> 디스크 절반은 복제본 → 비용은 높지만 속도와 안정성 모두 확보

## 🔑 핵심 개념 2: MS-SQL 라이선스 정책 (Bare Metal)

Bare Metal Server에서 MS-SQL을 사용할 때는 라이선스 정책에 주의해야 한다.

MS-SQL Server 2019 에디션별 주요 특징:

| 에디션 | 주요 특징 |
|--------|---------|
| **Enterprise** | Mission-critical 성능, 무제한 CPU 코어, 무제한 메모리, 페타바이트급 데이터 웨어하우징 |
| **Standard** | 중간 규모 애플리케이션용, 최대 24코어 CPU, 최대 128GB 메모리 |
| **Express / Web** | 소규모 웹/모바일 앱용, 무료 제공, 최대 16코어 CPU, 최대 64GB 메모리 |
| **Developer** | 개발/테스트 환경 전용, Enterprise 에디션의 모든 기능 무료 사용 |

> ⚠️ **시험 포인트**: Oracle 라이선스 정책과 동일한 방식으로 적용된다.

### Oracle 라이선스 에디션

Bare Metal 서버에서의 Oracle 라이선스도 알아두어야 한다.

| 에디션 | 주요 제약 |
|--------|---------|
| **Express Edition (XE)** | CPU 수와 무관, 단일 사용자만 가능, DB당 데이터 4GB까지, 메모리 1GB까지, 무료 |
| **Personal Edition** | CPU 수와 무관, 단일 사용자만 가능, 모든 Oracle 제품군 호환, Desktop급 장비용 |
| **Standard Edition One (SE1)** | 2 CPU까지 확장 가능한 서버, Min User 5, RAC 미지원, 400명 미만 조직에 적합 |
| **Standard Edition (SE)** | 최대 4 Processor(CPU) 또는 8 Core까지 제약, 클러스터링 지원, RAC 포함 |
| **Enterprise Edition (EE)** | Processor 수 제한 없음, Min User 25, RAC 옵션, 대기업용 |

## 🔑 핵심 개념 3: GPU Server

클라우드빵집이 AI 추천 시스템을 도입하기로 했다.  
"어떤 빵을 좋아하는 고객인지 딥러닝으로 예측해보자!"

GPU 서버는 **병렬 처리에 최적화된 GPU**를 장착한 서버로, 딥러닝을 위한 GPU 서버 팜을 제공한다.  
이미지 처리, 딥러닝 학습/추론, 과학 연산에 사용.

### GPU 서버의 핵심 특징 (⭐ 시험 단골!)

| 항목 | 내용 |
|------|------|
| **GPU 할당 방식** | **Pass Through** 방식 ← 이게 핵심! |
| **GPU 할당 단위** | **장(개) 단위** (Core 단위가 아님) |
| **장착 GPU** | NVIDIA T4, V100, A100 |

> ⚠️ **시험 포인트**  
> NCP GPU 서버는 **NVIDIA GRID 방식이 아닌 Pass Through 방식**!  
> - **NVIDIA GRID**: GPU를 쪼개서 여러 VM이 공유 (vGPU, Shared GPU)  
> - **Pass Through**: GPU 장 단위로 VM에 **전용 할당** → 성능 최대 (Dedicated GPU)

### GPU 서버 모델별 스펙

| 모델 | 서버 타입 | GPU 최대 수 | 특이사항 |
|------|----------|------------|---------|
| **T4** | VM | 최대 **2장** | - |
| **V100** | VM | 최대 **4장** | - |
| **A100** | **Bare Metal** | 최대 **8장** | 로컬 NVMe 디스크 기본 제공 |

### GPU 서버 지원 OS

| OS | 비고 |
|----|------|
| CentOS 7.3 | T4 미지원 |
| CentOS 7.8 | - |
| Ubuntu 16.04, 18.04 | P40 제외 |
| Ubuntu 20.04 | A100만 제공 |
| Windows 2016, 2019 | - |
| TensorFlow on Linux | CentOS 7.3(Classic 환경만), CentOS 7.8, Ubuntu 16.04 |

### NVIDIA GPU 아키텍처 역사

NCP GPU 서버에 사용되는 GPU들의 아키텍처 계보:

```
Tesla → Kepler → Maxwell → Pascal → Volta → Turing → Ampere
                           P4, P40   V100    T4       A100
                           P100
```

> 연도별로 2016~2020년에 걸쳐 발전해 왔으며, Training과 Inference 양쪽에 모두 활용된다.

### GPU 서버의 제약사항

| 항목 | 가능 여부 |
|------|----------|
| 스토리지 추가 | 가능 ✅ |
| 스토리지 크기 변경 | 가능 ✅ |
| SSD 스토리지 이용 | 가능 ✅ |
| 오토스케일링 | **불가** ❌ |
| 네트워크 인터페이스(NIC) 추가 | **이용 불가** ❌ |

## 서버 관련 자원 제한 (⭐ 시험 단골!)

| 리소스 | 기본값/제한 |
|-------|-----------|
| 서버 수 | 서버 타입에 따라 다름 (서버 타입별 서버 생성 수 제한 참조) |
| 서버당 최대 공인 IP 수 | **서버당 1개** |
| VPC당 최대 ACG 수 | **500개** |
| 서버당 최대 ACG 수 | **3개** |
| ACG당 최대 규칙 수 | **50개** |
| 최대 Network Interface 수 | **3개** (단, Micro-g3는 1개) |
| 서버 정지 가능 일수 | 1회 최대 **90일**, 12개월 누적 **180일** |

> ⚠️ **주의**: 공인 IP, 로드밸런서, 추가 디스크 등 서버에 연결된 다른 자원의 요금은 **서버 정지 시에도 정상적으로 청구**된다. 사용하지 않는 리소스는 반납 처리 필요.

## 서버 상태값

서버의 생명주기에 따른 상태 전환:

```
[생성 시]
생성 중 → 부팅 중 → 설정 중 → 운영 중

[정지 시]
운영 중 → 종료 중 → 정지

[시작 시]
정지 → 부팅 중 → 운영 중
```

## 서버 Operation 주요 기능

| 기능 | 설명 |
|------|------|
| **서버 스펙 변경** | 반드시 **정지된 상태**에서 변경 |
| **내 서버 이미지 생성** | 서버의 현재 상태를 이미지로 생성. 서버가 **정지 중** 또는 **운영 중** 상태에서 생성 가능 |
| **유사 서버 생성** | 선택한 서버와 동일한 스펙의 서버 생성 시 사용 (서버 이미지, 스토리지, 서버 타입, 요금제, Zone, 인증키, ACG 정보를 동일하게 적용) |
| **스토리지 생성** | OS 영역 50GB에 추가 디스크 생성. XEN 하이퍼바이저의 경우 최소 10GB ~ 최대 2TB, 서버 한 대에 최대 15개 추가 부착 가능 |
| **스토리지 용량 변경** | 용량 증설 가능. 서버 **정지 또는 연결 해제** 후 가능. 증설 후 볼륨 관리 툴 사용 필요 |
| **물리 배치 그룹 관리** | 서버들을 최대한 다른 클러스터에 분산. **Best-Effort 방식** (분산 결과 보장 안 함) |

## 📝 시험 대비 체크리스트

- [ ] Bare Metal: 하이퍼바이저 **없음**, 물리 서버 **단독** 사용 (다른 고객과 공유 안 함)
- [ ] Bare Metal: 스토리지 추가 **불가**, 크기 변경 **불가**, 오토스케일링 **불가**
- [ ] Bare Metal RAID: **RAID 1+0** → DBMS/안정성, **RAID 5** → 일반 웹 서비스
- [ ] Bare Metal 스펙: Single(8코어/128GB) ~ Dual Gold 6248R(48코어/512GB), A100(32코어/2TB/NVMe)
- [ ] Bare Metal IPMI 제공: 원격 하드웨어 전원 제어 및 모니터링 가능
- [ ] GPU: **Pass Through 방식** (NVIDIA GRID 아님!)
- [ ] GPU 할당 단위: **장(개) 단위** (Core 단위 아님)
- [ ] GPU VM: T4 최대 **2장**, V100 최대 **4장**
- [ ] GPU Bare Metal: A100 최대 **8장**, NVMe 기본 제공
- [ ] GPU 서버: 오토스케일링 **불가**, NIC **이용 불가**
- [ ] MS-SQL: Bare Metal에서 Oracle과 동일 라이선스 정책 적용
- [ ] 서버당 공인 IP **최대 1개**, ACG 최대 **3개**, ACG 규칙 **50개**
- [ ] NIC 최대 **3개** (Micro-g3는 1개)
- [ ] 서버 정지: 1회 최대 **90일**, 누적 **180일/12개월**
- [ ] 서버 스펙 변경: **정지 상태**에서만 가능
- [ ] 내 서버 이미지: **정지 중 또는 운영 중** 상태에서 생성 가능

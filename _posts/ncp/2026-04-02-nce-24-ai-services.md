## AI & Application 서비스 개요

NCP의 AI 서비스는 크게 세 가지로 분류된다.

| 구분 | 설명 |
|------|------|
| **AI 서비스** | 파파고, 챗봇, OCR, 스피치, 더빙, AiTEMS, 클로바스튜디오, 그린아이 등. 사용자가 콘솔에서 직접 빌드하고 다양한 구성 설정 가능 |
| **AI NAVER API** | REST API로 제공되는 AI 서비스 |
| **NCP Application** | Geolocation, SENS, Outbound Mailer 등 네이버 클라우드에서 제공하는 응용 서비스 |

### Application API 공통 인증 헤더 (시험 포인트!)

NCP Application API를 사용하려면 공통적으로 아래 헤더를 포함해야 한다.

| 헤더 | 설명 |
|------|------|
| `x-ncp-apigw-timestamp` | 1970년 1월 1일 00:00:00 UTC 기준 경과 시간(밀리초). API Gateway 서버와 시간 차가 **5분 이상** 나면 유효하지 않은 요청으로 간주 |
| `x-ncp-iam-access-key` | 포털 또는 SubAccount에서 발급받은 Access Key ID |
| `x-ncp-apigw-signature-v2` | Access Key ID와 매핑된 Secret Key로 암호화한 서명. **HMAC SHA256** 알고리즘 사용 |

## 서비스 1: CLOVA Chatbot — 대화형 AI 상담봇

### 핵심 개념

| 항목 | 내용 |
|------|------|
| **정의** | CS나 주문 시스템 같은 고객 대응을 로봇으로 대체하는 상품 |
| **기반 기술** | 네이버 NLU(자연어처리) 엔진 + 딥러닝 앙상블 모델 |
| **지원 언어** | 한국어, 일본어, 영어, 중국어, 태국어, 인도네시아어 (총 **6개 언어**) |
| **연동 메신저** | 라인, 톡톡, 페이스북 메신저 |
| **확장** | 학습을 통해 적절한 답변을 자동화. CSR, CSS와 연동하여 메신저뿐만 아니라 음성·채팅까지 확장 가능 |
| **관리 단위** | 도메인(Domain) 단위로 관리 |
| **빌더** | 학습을 위한 별도의 빌더 제공 |

### 자연어 처리 과정 (시험 포인트!)

```
[사용자 질문 입력]
       ↓
[Feature 추출] — 품사, 엔티티, 어미, 형태소 정보
       ↓
[여러 모델에 Feature 전달]
  ┌─ 모델A: 품사 중심 분석
  ├─ 모델B: 엔티티 중심 분석
  └─ 모델C: 기타 분석
       ↓
[앙상블 스코어 계산]
       ↓
[가장 정확한 답변 제공]
```

### 대화 구성 요소

```
도메인(Domain)
  └─ 대화(Dialog)
       ├─ 일반 대화: 발화 → 의도 파악 → 답변 (간단한 Q&A)
       └─ 태스크(Task): 슬롯 수집 → 복잡한 플로우 처리
```

**대화 vs 태스크 선택 기준**:
- **일반 대화**: 단순 Q&A. 대부분의 챗봇은 일반 대화만으로 서비스를 구성할 수 있음
- **태스크**: 예약·주문처럼 사용자와의 인터랙티브한 대화를 통해 여러 슬롯을 수집하고, 수집된 슬롯으로 특정 태스크를 수행하는 복잡한 플로우가 필요할 때

**태스크 흐름 설정**: 태스크와 일반 대화 사이의 흐름을 설정할 수 있다.

**대화 그룹 설정**: 대화 캔버스에서 대화 흐름을 분리된 페이지 공간에서 표시하기 위해 사용. 흐름이 연결된 작은 그룹 단위로 그룹핑하는 것을 권장.

### 챗봇 고급 설정 (시험 포인트!)

| 기능 | 설명 |
|------|------|
| **컨텍스트** | A 대화의 output 컨텍스트 = B 대화의 input 컨텍스트 → B 우선 매칭 |
| **네거티브** | 유사한 대화 간 차이를 모델에 학습시킴. 최대 **20개** 등록 |
| **웰컴 메시지** | 세션이 시작되었을 때 챗봇이 사용자에게 건네는 첫 메시지 |
| **실패 메시지** | 사용자의 질문에 응답할 답변을 찾지 못한 경우 대신 응답하는 메시지 |
| **무응답 메시지** | AiCall 도메인 전용. 전화 상황에서 설정된 음성 입력 대기 시간 동안 발화가 없을 때 사용자 발화를 유도하는 메시지 |
| **태스크 종료 키워드** | 태스크 진행 중 사용자가 강제로 태스크를 멈출 수 있는 키워드와 메시지 |
| **고정 메뉴** | 채팅 화면 하단에서 열고 닫을 수 있는 메뉴를 고정하는 기능 |
| **유사 답변** | 정확히 매칭 실패 시 '정확도 중간' 구간에 해당하는 대화를 캐로셀 형태로 보여주는 기능 |
| **피드백** | 챗봇의 답변을 사용자가 평가할 수 있는 기능 |

### 우선순위 규칙 (반드시 암기!)

> **객관식 버튼 연결 대화** > **컨텍스트 일치 대화** > **일반 대화**

## 서비스 2: CLOVA OCR — 문서/이미지 텍스트 추출

### OCR 처리 흐름

```
Pre-process → Area Detection → Classify → Text Recognition → Test/Output
(전처리)      (영역 검출)      (분류)      (글자 인식)        (결과 출력)

Template Classification (기존 작성 템플릿과 비교를 통한 행동 진품 검출)
Template Detection (지정된 영역 인식)
```

### 서비스 모델 비교 (시험 단골!)

| 항목 | **기본 (Basic)** | **프리미엄 (Premium)** |
|------|-----------------|----------------------|
| **인식 대상** | 활자체만 | 활자체 + **필기체** |
| **적합 문서** | 증명서, 고정 폼 양식 | 수기 신청서, 가입 양식, 금융 문서 등 |
| **멀티박스** | 제공 안함 | **제공** (싱글박스 템플릿을 조합하여 번호 인식 영역 등을 제어) |
| **체크박스** | 제공 안함 | **제공** (체크박스 형태의 인식 제공) |
| **필드 유형** | 제공 안함 | **제공** (인식값을 숫자로만 인식되도록 유형을 설정) |
| **인식 템플릿 레이아웃** | 제공 | 제공 (인식 영역을 설정) |

> 두 모델 모두 "주요 비즈니스 활용에 최적화된 고성능 OCR 인식 모델"을 제공

### API 호출 방식

```
Method: POST
Request URI: CLOVA OCR 빌더에서 생성된 API Gateway InvokeURL
             (도메인마다 고유 URL 생성)
```

### TEXT OCR vs Template OCR

| 구분 | **TEXT OCR** | **Template OCR** |
|------|-------------|-----------------|
| **경로** | `/general` | `/infer` |
| **용도** | 템플릿 정의 없이 이미지의 모든 텍스트를 인식 | 도메인에 배포된 템플릿이 포함된 이미지를 인식 |
| **언어 설정** | `ko` 또는 `ja` (미설정 시 `ko` 기본값) | 도메인의 언어 설정값이 기본값 |
| **matchedTemplate** | 반환 안됨 | 반환됨 |

> ICDAR2019 4개 분야 1위 기술력 보유

## 서비스 3: CLOVA Speech — 음성 인식 (STT)

### 특징

- **NEST 음성 인식 기술** 기반 미디어 내 음성 인식 서비스
- 음성 메모, 영상 자막 생성, 통화 녹취록 관리 등의 서비스 구현에 활용
- 인식 결과 수정을 위한 **Speech 빌더** 제공
- **화자 분리 기능** 제공, 인식 결과 신뢰도 학습을 통한 학습 알고리즘 개선
- 도메인 단위로 관리

### 인식 유형별 스펙

| 유형 | 이용 방식 | 인식 가능 언어 | 인식 가능 시간 | 인식 파일 크기 | 파일 포맷 |
|------|---------|--------------|--------------|--------------|--------|
| **장문 인식** | REST API | 한국어, 영어, 한/영 동시 인식, 일본어, 중국어(번체/간체) | 최대 2시간(sync) / 최대 6시간(Batch, async) | 최대 2GB(빌더) / 최대 20GB(API) | Audio: mp3, aac, ac3, ogg, flac, wav, m4a / Video: avi, mp4, mov, wmv, flv, mkv |
| **장문 인식** | gRPC | 한국어, 영어, 일본어 | 최대 100시간 | — | 16kHz (16bits per sample PCM) |
| **단문 인식** | REST API | 한국어, 영어, 일본어, 중국어 | 최대 60초 | 최대 10MB(빌더, API) | Audio: mp3, aac, ac3, ogg, flac, wav, m4a |

### 결과 파일 포맷

```
json, smi, CSV, xls, srt
```

### API 호출 3가지 방식

```
1. Object Storage 파일 업로드 방식
   POST /recognizer/object-storage

2. 외부 URL 인식 방식
   POST /recognizer/url

3. 로컬 파일 업로드 방식
   POST /recognizer/upload
```

각 도메인은 고유의 API 호출 URL을 제공한다. API로 호출한 경우 인식 결과 수정 에디터는 사용할 수 없다.

### 빌더 사용 방식

도메인 생성 후 CLOVA Speech 빌더를 사용하면 UI 환경으로 인식 작업을 요청할 수 있다. 원본 파일은 Object Storage에서 가져오거나 파일 업로드를 통해 설정하며, 작업 목록에서 처리 상태 및 결과를 확인할 수 있다. 빌더를 통한 작업 요청 시에는 인식 결과 편집이 제공된다.

### 주요 요청 파라미터 (시험 포인트!)

| 파라미터 | 설명 | 기본값 |
|---------|------|--------|
| `language` | 인식 언어 (ko-KR, en-US 등) | `ko-KR` |
| `completion` | sync/async 방식 | `async` |
| `sttEnable` | 음성 인식 사용 여부 | `true` |
| `wordAlignment` | 인식 결과에 word alignment 출력 | `true` |
| `fullText` | 전체 인식 결과 텍스트 출력 | `true` |
| `diarization.enable` | 화자 인식 여부 | `false` |
| `keywordExtraction.enable` | 키워드 추출 사용 여부 | `false` |
| `groupByAudio` | 변환 결과를 음원 파일별로 묶는 옵션 | `false` |

### 키워드 부스팅 제약사항 (시험 포인트!)

- 최대 **500건** 부스팅 가능
- 지원 문자: **한글, 영어, 숫자**만 가능
- `네`, `응`, `no` 같은 **1음절 단어는 부스팅 불가** (오인식 위험)
- 영문 인식 결과는 기본적으로 소문자로 변환되지만, 대문자 키워드 부스팅 요청 시 대문자로 치환
- 띄어쓰기 여부와 관계없이 부스팅 처리 (예: "클로바스피치"와 "클로바 스피치" 중 하나만 부스팅하면 됨)
- 키워드 길이 제약은 없으나 여러 단어가 조합된 긴 구문은 부스팅 영향을 받기 어려움

### 민감 키워드 디텍팅

- 인식 결과에 표시하지 않을 키워드 리스트를 `params.forbiddens` 필드에 포함
- 민감 키워드의 개수 및 길이 제한 없음
- 띄어쓰기, 영문 대소문자 모두 완전 일치한 경우에만 디텍팅 처리

## 서비스 4: CLOVA Voice — 음성 합성 (TTS)

### 특징

| 항목 | 내용 |
|------|------|
| **정의** | 텍스트 → 자연스러운 음성으로 변환하는 TTS 서비스 |
| **기술** | HDTS, NES(Natural End-to-End Speech Synthesis) |
| **음성 옵션** | **총 100가지** 합성음 제공 (성별, 연령대 포괄) |
| **최대 변환 글자** | **2000글자** |
| **지원 언어** | 한국어, 영어, 일본어, 중국어, 스페인어 등 다국어 |
| **지원 리전** | 한국, 미국, 싱가포르, 일본, 독일 |
| **감정 파라미터** | Volume, Speed, Pitch, Emotion 등 제공 |
| **출력 포맷** | MP3, WAV (WAV의 경우 샘플링 Rate 지원) |

### 서비스 플랜 (시험 포인트!)

| 구분 | **Standard** | **Premium** |
|------|-------------|-------------|
| **파일 다운로드** | O | O |
| **실시간 API 호출** | X | X |
| **재가공/재편집** | X | X |
| **개인/비영리 용도** | O | O |
| **키오스크/앱/웹 서비스** | X | △ (제휴 제안 필요) |
| **유료 판매** | X | △ |

> 두 플랜 모두 **실시간 API 호출 불가**, **재가공 불가**

## 서비스 5: CLOVA Dubbing — 미디어 파일 더빙

클라우드빵집 홍보팀이 반색했다. "영상에 성우 더빙을 자동으로 넣을 수 있다고요?"

CLOVA Dubbing은 다양한 미디어 파일에 음성을 더빙하는 서비스다.

### 특징

| 항목 | 내용 |
|------|------|
| **관리 단위** | **프로젝트** 단위로 관리 |

## 서비스 6: Outbound Mailer — 대량 이메일 발송

### 주요 기능

- 대량 메일 발송을 위한 메일 발송 상품
- 수신자 그룹 관리 및 조합 발송 가능
- 콘텐츠 개인화 메일 전송 가능
- 템플릿에 네이버 메일과 동일한 입력기인 **스마트에디터** 제공, 치환 태그 기능을 통해 변수 사용 가능
- 실시간 통계 조회 및 발송 이력 조회 (최대 **1년** 보관)
- 광고 메일 발송을 위한 템플릿 기능, 법적인 기능(제목에 광고 문구 삽입 및 수신 거부 기능) 제공
- REST API를 통해 메일 전송 가능

### 메일 기본 기능 제약사항

| 항목 | 제한 사항 |
|------|---------|
| **받는 사람 주소** | 최대 **1,000명**. 발송은 **30명씩** 나누어 순차 발송(콘솔 기준) |
| **참조 주소** | 최대 **30명** |
| **숨은 참조 주소** | 최대 **30명** |
| **파일 업로드 대량 발송** | 최대 **100,000개** 수신자 주소 입력 가능 |
| **첨부 파일 크기** | 개당 최대 **10MB**, 총 **20MB** (본문 이미지 파일도 포함) |

### 첨부 불가 확장자

```
bat, cmd, com, cpl, exe, scr, js, vbs, ocx, pif, msi, efi, fon, drv, dll, sys,
app, dmg, ipa, apk, jar, wsh, reg, hlp, lnk, wsf, scf, swf, cab
```
위 확장자 파일이 포함된 압축 파일(zip, rar 등)도 발송 실패할 수 있다.

### 치환 태그

**사용자 정의 치환 태그**: `${TAG_NAME}` 형식. 개수 제한 없이 입력 가능. 태그에 대체될 값이 하나라도 전달되지 않으면 발송 불가.

**Meta 정보 치환 태그**: `#{TAG_NAME}` 형식. 사전에 정의된 지정된 변수만 사용 가능.

| 태그명 | 설명 |
|------|------|
| `RECIPIENT_NAME` | 수신자 명 |
| `RECIPIENT_ADDRESS` | 수신자 이메일 주소 |
| `SENDER_NAME` | 발송자 이름 |
| `SENDER_ADDRESS` | 발송자 이메일 주소 |
| `UNSUBSCRIBE_URL` | 수신 거부 링크 URL |

### 발송 결과 코드 (시험 포인트!)

| 결과 코드 | 설명 |
|---------|------|
| `MAIL_SENT` | 발송 성공 |
| `RESENDING_MAIL` | 재발송 중 |
| `CANCELED_MAIL` | 발송 취소 |
| `SEND_BLOCK_ADDRESS` | 발송 차단된 주소로 요청하여 차단 |

| `CONNECTION_ABNORMAL` | 커넥션 이상으로 인한 **일시적** 발송 실패 |
| `MAILBOX_ABNORMAL` | 수신 측 메일함 이상으로 인한 **일시적** 발송 실패 |
| `MAILBOX_ERROR` | 수신 메일함 오류로 인한 발송 실패 |
| `NETWORK_ABNORMAL` | 네트워크 이상으로 인한 **일시적** 발송 실패 |
| `NETWORK_ERROR` | 네트워크 오류로 인한 발송 실패 |
| `NONEXISTENT_DOMAIN_ADDRESS` | 존재하지 않는 도메인 주소로 발송 요청 |
| `UNKNOWN_DOMAIN_ADDRESS` | 알 수 없거나 차단된 도메인이 포함된 주소로 발송 요청 |

| `RECEIVE_MAIL_SERVICE_ABNORMAL` | 수신 측 메일 서비스 이상으로 인한 **일시적** 발송 실패 |
| `RECEIVE_MAIL_SERVICE_ERROR` | 수신 측 메일 서비스 오류로 인한 발송 실패 |
| `SECURITY_AND_POLICY_ABNORMAL` | 보안 및 정책 이상으로 인한 **일시적** 발송 실패 |
| `SECURITY_AND_POLICY_ERROR` | 보안 및 정책 오류로 인한 발송 실패 |
| `CONTENT_HAS_TAG_FORM` | 콘텐츠(제목 및 본문) 내 치환 태그 존재로 인한 발송 실패 |
| `EMPTY_BODY_CONTENT` | 시스템 오류로 메일 본문을 설정하지 못해 발생한 발송 실패 |

## 서비스 9: GeoLocation — IP 기반 위치 정보

### 특징

- 사용자 IP를 통해 위치 정보를 제공하는 **국내 유일의 서비스**
- 고객 서버에서 질의한 IP 주소에 따른 지역 정보 DB 검색
- 접속 IP 기반으로 국내는 **동(洞)까지**, 해외는 **주(州)까지** 확인 가능
- REST API 기반으로 동작. 어뷰징 방지를 위해 **TimeStamp와 Signature**를 생성하여 요청

### 제공 항목

| 유형 | 세부 내용 |
|------|---------|
| **국가** | KR, JP, CN, US, DE, HK와 같은 국가 코드 |
| **지역 코드** | 한국의 경우 행정자치부 행정동 코드 |
| **지역명** | 도, 특별시/광역시, 주, 시, 군, 구, 동 |
| **좌표** | 위도, 경도 |
| **통신사** | 이용 통신사명 |

### 적용 사례

1. **네이버 검색 광고**: 검색 결과가 검색자의 현재 위치와 관련되는 경우 지역 관련 광고 노출
2. **네이버 날씨**: 사용자의 현재 위치에 대한 날씨를 관심 지역으로 제안
3. **네이버 지도**: 초기 화면이 사용자의 현재 위치로 설정
4. **네이버 동영상**: 특정 국가 제공 금지 미디어 콘텐츠를 사용자 위치에 따라 제공 여부 판단

### 요청 파라미터

| 파라미터 | 필수 | 타입 | 제약 사항 | 설명 |
|---------|------|------|---------|------|
| `ip` | Y | String | IPv4 주소 | 지역 정보를 알고자 하는 사용자 IP (예: 202.131.30.11) |
| `enc` | N | String | `utf8`(기본값) 또는 `euckr` | 응답 결과 인코딩 타입 |
| `ext` | N | String | `t` 또는 `f`(기본값) | 추가 정보 포함 여부 (`t`: 포함, `f`: 비포함) |
| `responseFormatType` | N | String | `xml`(기본값) 또는 `json` | 응답 결과 포맷 타입 |

### 응답 항목

| 항목 | 타입 | 설명 |
|------|------|------|
| `requestId` | String | API 요청 ID |
| `returnCode` | String | 정상 Code는 0, 이외 오류 코드 |
| `country` | String | 국가 코드 (ISO 3166 + EU, AP, A1, A2, O1) |
| `code` | String | 한국의 경우 행정자치부 행정구역 코드 |
| `r1` | String | 도, 광역시, 주 |
| `r2` | String | 시, 군, 구 |
| `r3` | String | 동, 면, 읍 |
| `lat` | Number | 위도 |
| `long` | Number | 경도 |
| `net` | String | 통신사 이름 |

| **CLOVA Voice** | 음성 합성 | 텍스트→음성 (TTS) | 100가지 음성, 최대 2000글자 |
| **CLOVA Dubbing** | 미디어 더빙 | 텍스트→더빙 음성 | 프로젝트 단위 관리, MyVoice |
| **Outbound Mailer** | 대량 이메일 | 이메일 발송 | 수신자 100,000명, 이력 1년 보관 |
| **Papago** | 번역 | 텍스트→텍스트 | 언어 감지 12개 언어, 높임말 번역 |
| **Search Trend** | 검색 분석 | API→통계 | 그룹 5개, 그룹당 20개, 일 1,000건 |
| **GeoLocation** | 위치 정보 | IP→위치 | 국내 유일, 동까지 확인 |

## 시험 직전 체크리스트

- [ ] CLOVA Chatbot 지원 언어 6개 외우기
- [ ] 대화 우선순위: 객관식 버튼 > 컨텍스트 > 일반
- [ ] 네거티브 최대 20개 제한
- [ ] CLOVA OCR Basic vs Premium 차이 (멀티박스, 체크박스, 필드 유형)
- [ ] TEXT OCR 경로 `/general`, Template OCR 경로 `/infer`
- [ ] Speech API 3가지 방식 (object-storage / url / upload)
- [ ] Speech 부스팅: 최대 500건, 한글·영어·숫자만, 1음절 불가
- [ ] CLOVA Voice: 100가지 음성, 최대 2000글자, 두 플랜 모두 실시간 API 호출 불가
- [ ] CLOVA Dubbing: 프로젝트 단위 관리, 동영상 최대 500MB/20분
- [ ] Outbound Mailer: 수신자 1,000명(콘솔), 파일 업로드 100,000명, 이력 1년
- [ ] Outbound Mailer 결과 코드: SEND_BLOCK_ADDRESS(차단 주소), SMTP_ABNORMAL(일시적), SMTP_ERROR(오류)
- [ ] Papago 언어 감지: 12개 언어 지원
- [ ] Search Trend: 그룹 5개, 그룹당 키워드 20개, 일 1,000건
- [ ] GeoLocation: 국내 유일, 동까지 확인 가능, TimeStamp+Signature 인증
- [ ] Application API 공통 헤더: timestamp, access-key, signature-v2 (HMAC SHA256)

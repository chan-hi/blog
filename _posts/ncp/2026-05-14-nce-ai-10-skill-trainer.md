## 핵심 요약

- 스킬 트레이너는 HyperCLOVA X와 외부 API를 연결하고, 특정 서비스에 필요한 여러 스킬을 모델에 학습시키는 도구입니다.
- 기존 챗봇이 시나리오를 많이 만들어야 하는 한계를, HyperCLOVA X의 한국어 이해와 API 호출 능력으로 보완합니다.
- 스킬셋, 스킬, API Spec, Manifest, 데이터 수집, 데이터 학습, 버전 관리가 핵심 흐름입니다.
- 라우터는 요청과 콘텐츠를 분류하고 필터링합니다.
- Function Calling은 언어 모델이 외부 함수나 API 호출을 제안하고, 사용자 애플리케이션이 실행하는 방식입니다.

## 챗봇 vs 스킬

기존 챗봇의 문제점:

- 시나리오를 하나하나 모두 만들어야 함
- 데이터 양이 많이 필요함
- 자연스러운 사용자 발화를 잘 이해하지 못함
- 수행할 수 있는 일이 제한적임

HyperCLOVA X SkillSet은 초거대 모델의 한국어 능력을 활용해 최소한의 데이터로 자연스러운 발화에 대응할 수 있습니다.

스킬로 가능한 일:

1. DB에 있는 지식을 활용해 답변
2. API를 이해하고 대신 수행
3. 멀티턴 대화 수행
4. 기존 챗봇이 수행하지 못한 기능 구현

## 튜닝 vs 스킬

| 구분 | 튜닝 | 스킬 |
|---|---|---|
| 방식 | 기존 모델 추가 학습 | 모델이 API/DB를 활용 |
| 정보 반영 | 학습 시점 데이터에 반영 | 실시간 업데이트 가능 |
| 적합 상황 | 특정 구조, 스타일, 판단 방식을 따라야 할 때 | 질문에 답하기 위해 특정 정보 조회가 필요할 때 |
| 예 | 답변 말투, 분류 기준, 법률 문서 스타일 | 날씨, 지역 검색, 상품 조회, 예약 가능 여부 |

스킬은 외부 API나 DB를 이용해 최신 정보를 가져오는 데 강합니다.

## 스킬 트레이너 사용 단계

1. 스킬셋 생성
2. 스킬 생성, API Spec
3. 스킬 생성, Manifest
4. 데이터 수집
5. 데이터 학습
6. 버전 관리

## 1단계. 스킬셋 생성

스킬셋은 특정 분야에 필요한 여러 스킬의 묶음입니다.

예:

- 여행 스킬셋: 항공권 조회, 항공권 예약, 호텔 조회
- 쇼핑 스킬셋: 상품 검색, 장바구니 관리

스킬셋 생성 시 입력 항목:

| 항목 | 설명 |
|---|---|
| 스킬셋 이름 | 영문자, 한글, 숫자, 공백 허용. 30자 이내 |
| 서비스 분야 | 금융, 교육, 여행 등. 없으면 기타 선택 |
| 스킬셋 설명 | 수행 가능한 역할을 구체적으로 작성. 400자 이내 |

스킬셋 설명은 라우터 생성에 활용되므로, 수행 가능한 역할을 명확히 써야 합니다.

## 2-1단계. 스킬 생성, API Spec

스킬은 스킬셋에 속하는 하나의 API 단위입니다.

OpenAPI Spec은 HTTP API를 소스코드나 문서 접근 없이도 검색하고 이해할 수 있게 만든 문서입니다. 모델이 적절한 URL을 생성하도록 하는 데 사용됩니다.

작성 요소:

| 항목 | 설명 |
|---|---|
| servers | API를 호출할 수 있는 도메인 |
| paths | 각 API 경로. get, post 등 메서드 포함 |
| summary | API 기능, 필요 정보, 응답 정보 요약 |
| parameters | API 파라미터 이름, 위치, 설명, 필수 여부 등 |

## 2-2단계. 스킬 생성, Manifest

Manifest는 해당 스킬을 통해 호출할 수 있는 API의 이름, 목적, 사용 방법 등을 입력하는 영역입니다.

| 항목 | 설명 |
|---|---|
| name_for_model | API 성격을 내포하고 간결한 이름. 다른 API와 겹치지 않게 작성 |
| description_for_human | API의 목적과 용도. 전체 API 역할과 내부 API 역할이 명확해야 함 |
| description_for_model | 각 API의 주요 사용 방법. 중요 파라미터와 주요 키워드를 서술 |

### Name for model 규칙

- 영문자와 숫자 허용
- 대문자로 시작
- 20자 이내
- 한글, 특수문자, 공백 불가
- 두 단어 이상이면 각 단어 첫 글자를 대문자로 작성
- 스킬 이름에 `API`라는 단어 사용 금지

## 3단계. 데이터 수집

데이터 수집은 다양한 사용자 발화를 모으고, 스킬 호출 과정과 최종 답변을 확인하는 단계입니다.

## 4단계. 데이터 학습, 5단계. 버전 관리

수집한 데이터를 모델에 학습하여 스킬셋 성능을 높입니다.

권장:

- 1개 스킬셋당 50~100개 데이터 수집 후 학습

데이터 학습 종류:

| 구분 | 설명 |
|---|---|
| 스킬 호출 학습 | 사용자 요청을 수행하기 위한 API URL과 파라미터를 모델이 정확히 생성하지 못한 경우, 이를 수정해 저장하고 학습 |
| 최종 답변 학습 | 스킬 호출 결과를 바탕으로 답변 생성 시 정보 누락이나 답변 스타일 변경이 필요할 때 수정해 저장하고 학습 |

## 신규 추가 기능: 라우터

라우터는 사용자의 요청과 콘텐츠를 분류하고 필터링하는 기능입니다. 입력된 요청을 적절한 도메인으로 분류하고, 설정된 필터 조건에 해당하는지 판별합니다.

라우터 동작 프로세스:

1. 사용자의 질문이 적절한 주제인지 도메인 판별
2. 도메인 내 허용된 주제인지 콘텐츠 필터 확인
3. 비윤리적이거나 민감한 내용인지 세이프티 필터 확인
4. 라우터를 통과하면 스킬을 사용해 API 데이터 확인
5. 라우터에서 허용되지 않은 내용이면 스킬을 실행하지 않고 거절 메시지 응답
6. 검색/조회 결과를 비교 및 요약하여 답변 생성
7. 사용자에게 전달

## 라우터 구성 요소

| 구성 요소 | 설명 |
|---|---|
| 도메인, Domain | 데이터를 분류하는 주요 카테고리 |
| 필터, Filter | 특정 도메인 내에서 적용되는 세부 규칙 |

### 세이프티 필터

| 필터 | 설명 |
|---|---|
| Unethical | 욕설, 범죄 같은 비윤리적 내용 감지 |
| Contentious | 정치/사회적으로 민감한 논쟁 이슈 또는 편향된 주장 감지 |

콘텐츠 필터 설명 작성 시, 부정 표현이나 지침 형태가 아닌 감지할 주제를 서술해야 합니다.

올바른 예: `해외 지역에 대한 주소 검색 요청입니다.`

잘못된 예: `해외 지역에 대한 주소 검색 요청은 처리할 수 없습니다.`

## Function Calling

Function Calling은 언어 모델이 외부 함수나 API를 호출하여 동적으로 정보를 가져오거나 작업을 수행할 수 있도록 설정하는 기능입니다.

흐름:

1. 사용자 입력
2. HyperCLOVA X가 사용자 입력과 함수 설명을 보고 함수 식별자와 파라미터 생성
3. 사용자 애플리케이션이 API 호출
4. API 응답을 모델에 전달
5. 모델이 최종 답변 생성

## Skill Trainer vs Function Calling

| 구분 | Skill Trainer | Function Calling |
|---|---|---|
| 개요 | LLM이 API를 호출해 외부 데이터를 가져오는 기능 | 단순 API 호출 외에도 스크립트 실행, 라이브러리 함수 호출 지원 |
| 연동 지원 | API | API, 스크립트 실행, 라이브러리 호출 |
| 호출 및 액션 수행 | Skill Trainer에 API 등록 후 Skill 시스템이 호출 | 사용자 애플리케이션에서 외부 API 실행 |
| 제공 환경 | CLOVA Studio UI로 API 등록, 학습, 호출 환경 제공 | Function Calling API 제공 |

핵심 차이는 `누가 실행하느냐`입니다. Skill Trainer는 CLOVA Studio의 스킬 시스템이 API 호출 환경까지 제공합니다. Function Calling은 LLM이 실행을 제안하고, 실제 실행은 사용자 애플리케이션에서 수행합니다.

## 시험 암기 포인트

- 스킬셋은 여러 스킬의 묶음이고, 스킬은 하나의 API 단위입니다.
- API Spec은 모델이 API 호출 구조를 이해하게 하고, Manifest는 API 이름/목적/사용법을 설명합니다.
- 스킬 학습은 스킬 호출 학습과 최종 답변 학습으로 나뉩니다.
- 라우터는 도메인 판별, 콘텐츠 필터, 세이프티 필터를 수행합니다.
- 콘텐츠 필터 설명은 부정 지시문이 아니라 감지할 주제를 서술해야 합니다.
- Skill Trainer는 CLOVA Studio UI 기반 API 등록/학습/호출 환경이고, Function Calling은 애플리케이션에서 함수를 실행합니다.

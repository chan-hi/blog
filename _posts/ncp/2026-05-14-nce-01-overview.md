## 핵심 요약

- 클라우드는 인터넷 기반 컴퓨팅으로, 온디맨드 방식과 종량과금제를 특징으로 한다.
- 클라우드 도입의 핵심 경제적 장점은 CAPEX를 OPEX로 대체하는 것이다.
- 서비스 모델은 IaaS / CaaS / PaaS / FaaS / SaaS 5가지로 구분된다.
- NCE는 4개 시험 (301, 302, 303, 305)으로 구성된다.

## 클라우드 컴퓨팅 개념

클라우드란 인터넷 기반의 컴퓨팅을 의미하며, 서버, 네트워크, 스토리지, 애플리케이션, 서비스 등 다양한 컴퓨팅 리소스에 **장소나 시간 제약 없이** 쉽고 빠르게 접근할 수 있게 해주는 환경이다.

### 핵심 경제적 장점 5가지

1. **CAPEX → OPEX 대체**: 사전 계획 선투자 없이 사용한 만큼 비용 지불
2. **규모의 경제에서 얻는 이점**: 대규모 클라우드의 규모의 경제 이점이 고객 혜택으로
3. **지속적인 기술 혁신 및 적용**: 새로운 서비스 상품 지속 제공
4. **빠른 프로비저닝 및 유연한 확장**: 몇 번의 클릭으로 단 몇 분 만에 서버 확보
5. **인프라 유지관리 부담 제거**: 사업의 핵심에 집중 가능

> CAPEX(Capital Expenditure): 미래 이윤 창출을 위한 선투자 비용  
> OPEX(Operating Expenditure): 운영에 드는 운영 비용

## 클라우드 서비스 모델

| 약어 | 풀네임 | 설명 |
|------|--------|------|
| **IaaS** | Infrastructure as a Service | Computing, Networking, Storage 등 IT 리소스를 서비스 형태로 사용 |
| **CaaS** | Container as a Service | 컨테이너 레벨의 기능들을 서비스 형태로 사용 |
| **PaaS** | Platform as a Service | 기본 인프라를 직접 관리하지 않으면서 다양한 플랫폼 소프트웨어를 서비스 형태로 사용 |
| **FaaS** | Function as a Service | 기능(function code) 단위로 실행. 서버 관리 없이 애플리케이션 수행 |
| **SaaS** | Software as a Service | 완제품 형태의 소프트웨어를 서비스 형태로 사용 |

### 관리 책임 분리

IaaS → PaaS → SaaS 순으로 클라우드 벤더 관리 영역이 넓어지고, 유저 관리 영역은 줄어든다.

| 계층 | On-Premises | IaaS | PaaS | SaaS |
|------|-------------|------|------|------|
| Application | 유저 | 유저 | 유저 | 벤더 |
| Runtime | 유저 | 유저 | 벤더 | 벤더 |
| OS | 유저 | 유저 | 벤더 | 벤더 |
| Virtualization | 유저 | 벤더 | 벤더 | 벤더 |
| Servers | 유저 | 벤더 | 벤더 | 벤더 |

## 배포 모델

| 모델 | 설명 |
|------|------|
| **Public Cloud** | 공용 클라우드. 누구나 접근 가능한 인터넷 기반 |
| **Private Cloud** | 특정 조직 전용. 보안성 높음 |
| **Hybrid Cloud** | Private + Public Cloud 결합 |
| **Multi Cloud** | Public Cloud 여러 개를 동시에 사용 |

## NAVER CLOUD PLATFORM 소개

### NCP의 6가지 핵심 강점

1. 세계적으로 인정받은 보안 기술
2. 다양한 서비스 플랫폼 기술
3. 국내 최대 IT 서비스의 안정적인 인프라 (정전·지진 대비, 3번 백업 분산 저장)
4. 쉽고 간편한 웹 기반 관리 도구
5. 글로벌 메이저급 인프라 품질
6. 24시간 365일 사용자 지원

### 보안 인증

| 인증명 | 설명 |
|--------|------|
| ISO 27001 | 글로벌 정보보호 관리체계 인증 |
| **CSA STAR** | 글로벌 클라우드 통제 수준 인증 (국내 최초 획득) |
| **CSAP** | 클라우드컴퓨팅 서비스 보안 인증 (공공 클라우드) |
| PCI DSS | 지불카드산업 정보보호 표준 인증 |

## 데이터센터 & 리전 구조

- **리전**: 국가 단위의 데이터센터 그룹
- **존(Zone)**: 국가 단위 리전 내에 물리적으로 분리되어 존재하는 데이터센터 및 네트워크

> **멀티 존을 제공하는 리전**: **한국(KR), 싱가포르(SG), 일본(JP) 리전만** 멀티 존 구성 제공

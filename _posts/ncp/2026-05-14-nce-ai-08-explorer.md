## 핵심 요약

- 익스플로러는 CLOVA Studio에서 유용한 특화 도구와 API를 확인하고 활용하는 영역입니다.
- 주요 기능은 데이터 확장, 일괄 생성, 토큰 계산기, 슬라이딩 윈도우, 문단 나누기, 요약, 임베딩, 리랭커, RAG Reasoning입니다.
- 시험에서는 각 도구가 `무엇을 입력받고 무엇을 해결하는지`를 구분해야 합니다.

| 구분 | 기능 |

## 데이터 확장

데이터 확장은 사용자가 업로드한 데이터 샘플을 원하는 양만큼 확장하고 관리하는 도구입니다. 시드 데이터셋을 업로드하면 언어 모델이 패턴을 분석해 비슷한 유형의 데이터를 생성합니다.

### 활용 단계

1. 모델 엔진 선택
2. 시드 데이터셋 업로드
3. 원하는 데이터 수 입력
4. 실행
5. `내 작업`에서 다운로드

### 입력 데이터

튜닝과 동일하게 `Text(입력값) - Completion(결과값)` 형태로 구성된 데이터가 필요합니다. 원문 기준 최소 10건 이상 포함해야 합니다.

### 유의 사항

- 계정별로 동시에 1건만 수행할 수 있습니다.
- 데이터 1건 생성에 약 10초가 소요되며 시스템 환경에 따라 달라질 수 있습니다.
- 작업 시작 후 중단하면 진행 구간에 따라 과금될 수 있습니다.
- 작업 완료 후 최대 7일 동안 다운로드할 수 있습니다.
- 개인정보가 포함된 데이터 업로드에 따른 책임은 사용자에게 있습니다.

### 적합한 작업

데이터 확장은 답변이 정해져 있는 정답형 작업보다 새로운 문장을 생성하는 창작형 작업에 더 적합합니다.

## 일괄 생성

일괄 생성은 사용자가 업로드한 대량 작업을 일괄 처리하고 결과물을 관리하는 도구입니다.

### 활용 단계

1. 모델 엔진 선택
2. 시드 데이터셋 업로드
3. 프롬프트 템플릿 입력
4. 실행
5. `내 작업`에서 다운로드

### 입력 데이터

일괄 생성 데이터셋은 `Text(입력값)`에 해당하는 내용만 작성된 데이터로 10건 이상 포함해야 합니다. 튜닝 모델을 선택했다면 학습 데이터셋과 동일한 입력 구조로 구성해야 합니다.

### 유의 사항

- 계정별로 동시에 1건만 수행할 수 있습니다.
- 데이터 1건 생성에 약 10초가 소요됩니다.
- 작업 시작 이후 중단하면 진행 구간에 따라 과금될 수 있습니다.

## 토큰 계산기 API

토큰 계산기 API는 입력한 문장의 토큰 수와 글자 수를 계산합니다.

활용 목적:

- 업로드 데이터의 토큰 수 확인
- 최적의 토큰 수 찾기
- 효율적인 프롬프트 작성
- Chat Completions API 입력 전 토큰 예측

토큰 계산 기준:

- 입력된 한 글자 기준 계산
- 띄어쓰기와 줄바꿈도 1토큰 계산
- 한글 모델 기준 결과이며 외국어 입력 시 플레이그라운드 토큰 수와 다를 수 있음

종류:

| 종류 | 설명 |
|---|---|
| 토큰 계산기, 챗 | HyperCLOVA X 모델 및 Chat Completions API 입력 문장의 토큰 수 계산 |
| 토큰 계산기, 임베딩 v2 | 임베딩 v2 API의 bge-m3 모델에서 입력 문장의 토큰 수 계산 |

## 슬라이딩 윈도우 API

슬라이딩 윈도우 API는 챗 모드에서 프롬프트와 결과값을 모델이 처리 가능한 최대 토큰 수 안으로 조절하여 대화가 끊기지 않게 돕습니다.

### 필요한 이유

챗 모드에서는 사용자와 어시스턴트의 대화가 모델 최대 토큰 수를 초과하면 더 이상 새 대화를 생성하지 못합니다. 슬라이딩 윈도우 API는 오래된 대화 턴을 삭제해 이 상황을 방지합니다.

### 동작 방식

입력된 대화 내역의 총 토큰 수를 `A`, 새롭게 생성될 대화의 최대 생성 토큰 수를 `B = maxTokens`, 모델이 처리 가능한 최대 토큰 수를 `X`라고 할 때:

```text
A + B > X
```

이면 Chat Completions API는 새 대화를 생성하지 않습니다.

슬라이딩 윈도우 API는 초과된 토큰 수 `A + B - X`만큼 기존 대화 중 오래된 대화 턴부터 삭제합니다. 일부 문장만 자르지 않고, 대화 턴 단위로 삭제합니다.

주의:

- 챗 모드 모델, Chat Completions API에 대해서만 작동
- 슬라이딩 윈도우 API 결과가 그대로 Chat Completions API에 입력되도록 순서 설정 필요
- modelName, maxTokens 등 설정값은 함께 사용하는 Chat Completions API와 동일하게 설정
- 시스템 지시문 다음 대화, 즉 가장 먼저 입력된 대화 턴부터 삭제

## 문단 나누기 API

문단 나누기 API는 문장 간 유사도를 계산하여 주제 단위의 단락을 구분합니다. RAG에서 긴 문서를 임베딩 가능한 청크로 나눌 때 중요합니다.

### 주요 파라미터

| 파라미터 | 설명 |
|---|---|
| text | 분할할 텍스트 |
| alpha | 분할 정도 조절 |
| segCnt | 나눌 문단 수. 1 이상, -1이면 모델이 최적 문단 수로 분리 |
| postProcess | 후처리 여부 |
| postProcessMaxSize | 문단에 포함되는 최대 문자열 글자 수. 기본값 1,000 |
| postProcessMinSize | 문단에 포함되는 최소 문자열 글자 수. 범위는 0~postProcessMaxSize |

요청 URL:

```text
POST /v1/api-tools/segmentation
```

## 요약 API

요약 API는 긴 문서를 맥락에 맞게 문단으로 구분하고, 각 문단을 요약합니다.

### 주요 파라미터

| 파라미터 | 설명 |
|---|---|
| texts | 요약할 텍스트 배열 |
| autoSentenceSplitter | 자동 문장 분리 여부. 기본값 true |
| segCount | 분리할 문단 수. -1이면 모델이 최적 문단 수로 분리 |
| segMaxSize | 한 문단의 최대 글자 수. 1~3000자, 한글 기준 공백 포함 |
| segMinSize | 한 문단의 최소 글자 수. 0~segMaxSize |
| includeAiFilters | AI Filter 포함 여부. 기본값 false |

요청 URL:

```text
POST /v1/api-tools/summarization/v2
```

## 임베딩 API

임베딩 API는 입력 텍스트를 숫자 형태의 벡터값으로 변환합니다.

임베딩의 장점:

- 계산이 빠르고 효율적
- 데이터 간 관계 파악이 쉬움
- 원하는 정보 추출에 활용 가능
- 문서의 의미와 특징을 수치적으로 표현
- 벡터 공간에 매핑하여 유사성 계산 가능

RAG에서는 문서와 질문을 모두 벡터화하여 유사도를 계산합니다.

### 임베딩 모델 비교

| 도구명 | 모델명 | 최대 토큰 수 | 벡터 차원 | 권장 거리 지표 | 특징 |
|---|---|---:|---:|---|---|
| 임베딩 | clir-emb-dolphin | 500 | 1024 | IP, Inner/Dot/Scalar Product | 랭체인 기본 설정, 도메인과 무관한 범용성, 큰 범위 유사도 |
| 임베딩 | clir-sts-dolphin | 500 | 1024 | Cosine Similarity | 두 문장의 의미적 유사도 비교, 세부 범위 유사도 |
| 임베딩 v2 | bge-m3 | 8192 | 1024 | Cosine Similarity | 장문 단위 클러스터링, 한/영/일 포함 100개 언어 지원, 오픈소스 모델 |

clir-emb-dolphin은 회의록, 논문, FAQ 등 문단 단위 클러스터링에 유리합니다. clir-sts-dolphin은 문장 단위 유사성 분석과 텍스트 분류에 유리합니다. bge-m3는 최대 입력 토큰 수가 커 장문 처리에 유리하고, 문장의 유사성과 맥락 해석에 특화되어 있습니다.

## 시험 암기 포인트

- 데이터 확장은 기존 Text-Completion 샘플을 늘리는 도구입니다.
- 일괄 생성은 대량 입력을 같은 프롬프트 템플릿으로 한 번에 처리합니다.
- 토큰 계산기는 입력 토큰 수를 계산해 비용/한도/프롬프트 효율화에 사용합니다.
- 슬라이딩 윈도우는 오래된 대화 턴을 제거해 토큰 초과를 방지합니다.
- 문단 나누기는 주제 단위 분할, 요약은 문단 구분 후 요약, 임베딩은 벡터 변환입니다.
- 임베딩 모델별 거리 지표: clir-emb-dolphin은 IP, clir-sts-dolphin과 bge-m3는 Cosine입니다.

## 문제 1

네이버 클라우드 플랫폼에는 다양한 상품들이 카테고리로 그룹화되어 있습니다. "CloudHadoop" 상품은 어떤 카테고리에 속해 있을까요?

**보기:**
- 1) Compute
- 2) Database
- 3) Analytics
- 4) AI-Application Service

> **정답: Analytics**
>
> **해설:** Cloud Hadoop은 빅데이터 분석을 위한 상품으로 Analytics 카테고리에 속합니다. Compute는 서버 관련 상품, Database는 DB 관련 상품 카테고리입니다.

## 문제 2

다음 중, 사용자가 Agent를 설치해야지만 사용할 수 있는 서비스가 아닌 것은?

**보기:**
- 1) Cloud Insight
- 2) System Security Checker
- 3) CLA
- 4) 백업 서비스

> **정답: Cloud Insight**
>
> **해설:** Cloud Insight는 NCP 인프라의 기본 메트릭을 에이전트 없이 수집할 수 있습니다. 단, 상세 메트릭(OS 레벨)은 에이전트가 필요합니다. System Security Checker, CLA, 백업 서비스는 에이전트 설치가 필수입니다.

- 3) 일본
- 4) 독일

## 문제 5

서버 정지가 필요한 경우가 아닌 것은? (복수정답)

**보기:**
- 1) 서버 이미지 생성
- 2) 서버 스펙 변경
- 3) 서버 반납
- 4) 공인 IP 부여

> **정답: 서버 이미지 생성, 공인 IP 부여**
>
> **해설:** VPC 환경에서는 서버가 운영 중인 상태에서도 내 서버 이미지 생성이 가능합니다. 공인 IP 부여도 서버 정지 없이 가능합니다. 서버 스펙 변경과 서버 반납은 정지 상태에서 수행해야 합니다.

## 문제 6

오토스케일링 설정시 오토 스케일링 그룹(ASG)에서 설정 가능한 내용이 아닌 것은?

**보기:**
- 1) 오토스케일링으로 생성될 서버의 OS
- 2) 오토스케일링에서 사용할 Launch Configuration
- 3) 서버 증가, 감소 정책
- 4) 일정 설정

> **정답: 오토스케일링으로 생성될 서버의 OS**
>
> **해설:** 오토스케일링으로 생성될 서버의 OS는 Launch Configuration에서 설정합니다. ASG(Auto Scaling Group)에서는 Launch Configuration, 증감 정책, 일정 등을 설정할 수 있습니다.

## 문제 7

다음이 설명하는 내용은 Auto Scaling의 어떤 항목인가?
"새로운 서버가 생성 되었다고 해도, Init-Script 실행이나 업데이트 설치 등의 이유로 실제 서비스를 수행할 수 있을 정도로 준비되기까지는 시간이 소요될 수 있습니다."

**보기:**
- 1) 쿨다운
- 2) 헬스체크
- 3) 기대용량
- 4) Policy

> **정답: 쿨다운**
>
> **해설:** 쿨다운(Cooldown)은 새 서버가 생성된 후 실제 서비스 준비가 될 때까지 모니터링 이벤트 알람을 무시하도록 설정하는 유예 기간입니다.

## 문제 8

Data Teleporter에 대한 설명으로 잘못된 것은?

**보기:**
- 1) Data Teleporter 한 대당 100TB의 저장용량을 가짐
- 2) 별도의 요청이 없다면 최초 대여 후 60일 이내에 반환되어야 함
- 3) 고객이 신청한 암호화 키로 256비트 암호화됨
- 4) Linux/Windows Server 모두 NFS 또는 CIFS로 연결 가능

> **정답: 별도의 요청이 없다면 Data Teleporter는 최초 대여 후 60일 이내에 네이버 클라우드 플랫폼으로 반환되어야 합니다.**
>
> **해설:** 실제 반환 기한은 30일입니다. Data Teleporter는 대용량 데이터를 물리적 이동식 스토리지로 이전하는 상품입니다.

## 문제 9

다음 중, NAS에 대한 설명 중 잘못된 것은?

**보기:**
- 1) NAS는 임계치 설정에 따른 이벤트 알림 기능을 제공한다
- 2) NAS는 일정한 스케줄에 따라 자동 스냅샷을 완성한다
- 3) NAS는 리눅스 서버에서 공유 가능한 CIFS 프로토콜을 제공한다
- 4) 네이버클라우드플랫폼에서는 NAS 생성 시 마운트포인트 정보를 제공한다

> **정답: NAS는 리눅스 서버에서 공유 가능한 CIFS 프로토콜을 제공한다**
>
> **해설:** CIFS(Common Internet File System)는 Windows 서버를 위한 프로토콜입니다. Linux 서버는 NFS(Network File System) 프로토콜로 NAS를 마운트합니다.

## 문제 10

베어메탈에 대한 설명 중 틀린 것은?

**보기:**
- 1) 3가지 스펙을 제공하고 있다
- 2) OS로 Oracle Linux도 제공한다
- 3) 디스크 구성시 RAID 방식을 선택할 수 있다
- 4) 내 서버 이미지, 스냅샷, 추가 스토리지 기능은 제공하지 않는다

> **정답: 3가지 스펙을 제공하고 있다**
>
> **해설:** 베어메탈 서버는 3가지 이상의 스펙을 제공합니다. Oracle Linux, RAID 1+0/RAID 5 선택, 내 서버 이미지/스냅샷 미제공은 모두 올바른 설명입니다.

## 문제 11

Bare Metal 서버에 대한 설명 중 틀린 것은?

**보기:**
- 1) RAID 1 구성방식만 제공한다
- 2) 요금제는 시간 요금제만 제공한다
- 3) Init Script 사용이 가능하다
- 4) Oracle Linux가 제공된다

> **정답: RAID 1 구성방식만 제공한다**
>
> **해설:** 베어메탈 서버는 RAID 1+0과 RAID 5 방식을 제공합니다. 요금은 시간 요금제만 제공하며, Init Script와 Oracle Linux는 지원됩니다.

## 문제 12

다음 중, 네이버 클라우드 플랫폼 서버에 스토리지 추가와 관련하여 틀리게 설명한 것은?

**보기:**
- 1) Linux의 LVM, Window는 동적디스크 할당을 통해 여러 개의 디스크를 하나의 볼륨으로 묶을 수 있습니다
- 2) 서버에 연결된 디스크를 다른 서버로 옮길 수 있습니다
- 3) 사용 중인 디스크의 크기가 부족할 경우, 최대 15개의 스토리지를 추가할 수 있습니다
- 4) 스토리지 당, 100GB, 500GB, 1TB의 단위의 용량만 추가할 수 있습니다

> **정답: 스토리지 당, 100GB, 500GB, 1TB의 단위의 용량만 추가할 수 있습니다**
>
> **해설:** NCP Block Storage는 10GB 단위로 최대 2TB까지 자유롭게 용량을 설정할 수 있습니다. 특정 단위로만 제한되지 않습니다.

## 문제 14

다음 스토리지 상품 특징에 대한 설명으로 맞는 것은?

**보기:**
- 1) NAS는 10GB, 100GB, 500GB 등 다양한 단위로 스토리지 증가가 가능하다
- 2) Block Storage와 NAS는 모두 공유 가능한 스토리지이다
- 3) NAS는 동적으로 볼륨 사이즈 조정이 가능하고, 자동 스냅샷 및 이벤트 알람 기능을 제공한다
- 4) NAS는 Windows 서버만 이용할 수 있는 스토리지다

> **정답: NAS는 동적으로 볼륨 사이즈 조정이 가능하고, 자동 스냅샷 및 이벤트 알람 기능을 제공한다**
>
> **해설:** NAS는 여러 서버가 동시에 접근 가능한 공유 스토리지로, Linux/Windows 모두 지원하며 자동 스냅샷과 이벤트 알림 기능을 제공합니다. Block Storage는 단일 서버 전용입니다.

## 문제 16

CDN+ 설정 시 설정 항목에 대한 설명 중 올바르지 않은 것은?

**보기:**
- 1) Purge: CDN 캐시 서버에 저장되어 있는 콘텐츠를 삭제하는 기능
- 2) Secure Token: 토큰 기반의 인증으로 허용된 접근에만 콘텐츠를 전달
- 3) Ignore Query String: 콘텐츠가 지정된 도메인에만 제공되며 다른 사이트에서 참조되는 것을 방지
- 4) Access Log: CDN 사용 로그를 확인할 수 있는 기능

> **정답: Ignore Query String: 콘텐츠가 지정된 도메인에만 제공되며 다른 사이트에서 참조되는 것을 방지**
>
> **해설:** 설명에 해당하는 기능은 Referrer Domain입니다. Ignore Query String은 URL의 GET 파라미터("?id=1234" 등)를 제거한 후 원본 서버에 요청하는 기능입니다.

## 문제 17

다음 상품 중 상품 생성 시 CDN을 함께 생성하는 기능이 없는 상품은?

**보기:**
- 1) VOD Transcoder
- 2) Image Optimizer
- 3) Live Station
- 4) VOD Station

> **정답: VOD Transcoder**
>
> **해설:** VOD Transcoder는 동영상 변환 도구로 CDN 자동 생성 기능을 제공하지 않습니다. Image Optimizer, Live Station, VOD Station은 생성 시 CDN을 함께 생성하는 기능을 제공합니다.

## 문제 18

다음 중 VPC에서 사용 가능한 IP 대역은 무엇인가요?

**보기:**
- 1) 10.0.0.0/12
- 2) 172.17.0.0/16
- 3) 192.168.0.0/8
- 4) 192.168.1.0/30

> **정답: 172.17.0.0/16**

## 문제 19

네이버 클라우드 플랫폼의 VPC 환경에서 ACG에 대한 설명으로 옳은 것은?

**보기:**
- 1) 프로토콜은 TCP, UDP만 지원한다
- 2) Subnet당 100개 생성이 가능하다
- 3) 서버와의 맵핑 정보 변경이 가능하다
- 4) Rule에 우선순위가 있다

> **정답: 서버와의 맵핑 정보 변경이 가능하다**
>
> **해설:** ACG는 TCP, UDP, ICMP 프로토콜을 지원하며, VPC당 최대 500개 생성 가능합니다. ACG는 우선순위 없이 모든 룰이 적용되며(White List), 서버에 맵핑된 ACG 정보는 변경 가능합니다.

## 문제 20

Image Optimizer에 대한 설명으로 틀린 것을 고르시오.

**보기:**
- 1) 이미지 리사이즈 기능을 제공한다
- 2) 이미지 크롭 기능을 제공한다
- 3) 이미지 자동회전 기능을 제공한다

## 문제 21

애플리케이션 로드밸런서의 특징으로 틀린 것은?

**보기:**
- 1) HTTP, HTTPS 프로토콜 지원
- 2) 고정 IP 제공
- 3) URL 기반 분기 가능
- 4) 별도의 설정 없이 Client IP 확인 가능

> **정답: 별도의 설정 없이 Client IP 확인 가능**
>
> **해설:** Application Load Balancer는 Proxy 방식으로 동작하므로, 실제 Client IP를 확인하려면 X-Forwarded-For 헤더를 통한 별도 설정이 필요합니다. 고정 IP는 Network LB에서 제공됩니다.

## 문제 22

네이버 클라우드 플랫폼의 VPC 환경에서 ACG 룰이 적용되는 장치는?

**보기:**
- 1) 서버
- 2) 서버 NIC
- 3) IP
- 4) Subnet

> **정답: 서버 NIC**
>
> **해설:** VPC 환경에서 ACG는 서버의 NIC(Network Interface Card) 단위로 적용됩니다. Subnet 단위로 적용되는 것은 NACL(Network ACL)입니다.

## 문제 23

네이버 클라우드 플랫폼 VPC의 Subnet에 대한 설명으로 틀린 것은?

**보기:**
- 1) Private Subnet과 Public Subnet이 있습니다
- 2) 서버들은 Private Subnet과 Public Subnet 모두 위치 시킬 수 있습니다
- 3) 공인 IP가 필요한 로드밸런서는 Public Subnet에 위치합니다
- 4) Private Subnet에 있는 서버들은 공인 IP 부여를 할 수 없습니다

> **정답: 서버들은 Private Subnet과 Public Subnet 모두 위치 시킬 수 있습니다**
>
> **해설:** Private Subnet에 있는 서버는 공인 IP를 부여받을 수 없습니다. 공인 IP가 필요한 서버는 Public Subnet에 위치해야 합니다.

## 문제 24

다음 상품 중 이벤트 설정이 불가능한 상품은? (복수정답)

**보기:**
- 1) Server
- 2) NAS
- 3) Object Storage
- 4) CDN+

>

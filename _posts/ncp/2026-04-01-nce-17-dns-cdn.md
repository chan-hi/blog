## 이야기: 도메인을 사다

클라우드빵집이 `cloudbakery.co.kr` 도메인을 샀다.
그리고 전국 고객들이 빠르게 이미지를 볼 수 있도록 CDN도 붙이기로 했다.

> "DNS가 정확히 뭘 하는 거야?"

재현이 설명했다.

> "전화번호부야. 사람들은 'cloudbakery.co.kr'을 입력하는데,
> 실제 서버는 IP로 찾아야 해. DNS가 도메인 → IP로 번역해줘."

그런데 이미지 파일이 수백만 개였다. 원본 서버 한 대로는 감당이 안 됐다.

> "웹 서버를 통해 배포하게 되면 서버 용량이 기하급수적으로 커져야 해.
> CDN이 바로 이런 대규모 트래픽에 효과적으로 대응하기 위한 서비스야."

이미지, 게임 설치 파일, 미디어 영상 등 다양한 대용량 콘텐츠를 사용자 가까운 엣지 서버에 캐싱해 두면, 콘텐츠 사용량이 급증하더라도 안정적으로 전송해 서비스 연속성을 유지할 수 있다.

## 🔑 핵심 개념 1: DNS 기본 개념

**DNS (Domain Name System)** = 도메인 이름을 IP 주소로 변환하는 시스템.

```
[DNS 조회 과정]
사용자: cloudbakery.co.kr 입력
        ↓
DNS 서버: cloudbakery.co.kr = 110.10.20.30 (IP)
        ↓
브라우저: 110.10.20.30 서버로 접속
```

|------|------|

## 🔑 핵심 개념 3: DNS 레코드 타입 9가지 (⭐⭐ 최빈출!)

| 레코드 | 역할 | 특이사항 |
|--------|------|---------|
| **A** | 도메인 → **IPv4** 주소 매핑 | 가장 기본 |
| **AAAA** | 도메인 → **IPv6** 주소 매핑 | A의 IPv6 버전 |
| **CNAME** | 도메인에 **별명(Alias)** 지정 | **다른 레코드와 공존 불가** |
| **MX** | 메일 수신 **메일 서버** 지정 | **Preference 값 작을수록 우선순위 높음** |
| **SPF** | 발송자와 메일 서버 정보 일치 검증 → **스팸/사칭 방지** | TXT 레코드 형태로도 사용 |
| **TXT** | 도메인에 텍스트 정보 추가 | 유효성 판단, SPF와 유사 용도 |
| **SRV** | 서비스를 호스팅하는 서버의 **위치(호스트명, 포트번호)** 저장 | - |
| **CAA** | 인증서 발급 가능한 **인증기관(CA) 지정** → 가짜 인증서 방지 | - |
| **DS** | **DNSSEC**에서 서명 있는 영역 식별 데이터 저장 | - |

> ⚠️ **CNAME 주의**: CNAME은 같은 도메인에 다른 레코드(A, MX 등)와 **공존 불가!**

### 자주 나오는 문제 패턴

```
Q: "메일 서버를 지정하는 레코드 타입은?" → MX
Q: "MX 레코드에서 우선순위가 높은 것은?" → Preference 값이 작은 것
Q: "도메인에 별명을 지정하는 레코드는?" → CNAME
Q: "가짜 인증서 발급을 막는 레코드는?" → CAA
Q: "스팸 메일 방지를 위한 레코드는?" → SPF (또는 TXT)
```

## 🔑 핵심 개념 4: CDN (Content Delivery Network)

**CDN** = 콘텐츠를 사용자 가까운 캐시 서버에 저장하여 빠르게 전달하는 서비스.

```
[CDN 없을 때]
서울 사용자 → 원본 서버 (부산) → 시간 걸림
뉴욕 사용자 → 원본 서버 (부산) → 엄청 오래 걸림

[CDN 있을 때]
서울 사용자 → 서울 CDN 캐시 서버 → 빠름!
뉴욕 사용자 → 뉴욕 CDN 캐시 서버 → 빠름!
```

### Cache Hit vs Cache Miss (⭐)

| 상태 | 설명 | 동작 |
|------|------|------|
| **Cache Hit** | 캐시 서버에 콘텐츠가 **있음** | 캐시에서 바로 응답 (빠름!) |
| **Cache Miss** | 캐시 서버에 콘텐츠가 **없음** | 원본 서버에 요청 후 캐시에 저장 |

```
[처음 요청 — Cache Miss]
사용자 → CDN 캐시 서버 (없음) → 원본 서버 → CDN 캐시 저장 → 응답

[이후 요청 — Cache Hit]
사용자 → CDN 캐시 서버 (있음!) → 즉시 응답 ⚡
```

## 🔑 핵심 개념 5: NCP Global Edge 주요 기능

NCP의 CDN 서비스는 **Global Edge**라는 이름으로 제공된다. 네이버 클라우드 플랫폼에서 제공하는 다양한 서비스뿐 아니라 **외부 원본을 포함한 HTTP 지원 환경의 모든 엔드포인트**를 원본으로 지정할 수 있다.

| 기능 | 내용 |
|------|------|
| **엣지 유형** | 한국/일본 엣지, 글로벌 성능우선(네이버+파트너), 글로벌 가용성우선(파트너) |
| **룰빌더** | 캐시 제어, 헤더 재정의, URL 리다이렉션 및 리라이트 설정 가능 |
| **최신 프로토콜** | **HTTP/2**, **TLSv1.3** 지원 (네트워크 연결 시간 단축 및 안정성 강화) |
| **보안** | Secure-Token, **JWT 기반 보안 URL** |
| **접근 제어** | IP 기반, **GEO(지역) 기반**, Referer 기반 접근 제어 |
| **압축** | 최신 압축 알고리즘 **brotli** 지원 |
| **대용량 전송** | **byte range** 지원 (대용량 파일 분할 다운로드) |

### 엣지 유형 상세

- **한국 또는 일본 선택**: 해당 국가에 위치한 엣지 서버를 통해서만 콘텐츠 캐싱 및 전송
- **글로벌 (성능우선)**: 네이버 클라우드 네트워크와 파트너 네트워크를 통해 세계에 위치한 모든 엣지 서버를 통해 콘텐츠 캐싱 및 전송
- **글로벌 (가용성우선)**: 파트너 네트워크만을 통해 세계에 위치한 모든 엣지 서버를 통해 콘텐츠 캐싱 및 전송

## 🔑 핵심 개념 6: Global Edge 캐시 옵션 (⭐ 시험 출제 포인트!)

### 캐시 동작 설정

| 옵션 | 설명 |
|------|------|
| **캐시** | 엣지 서버에 개체를 캐싱 |
| **바이패스 캐시** | 개체를 캐싱하지 않고 오리진 서버로 직접 요청 |
| **원본 Cache-Control 헤더 우선** | 오리진 서버에 설정된 Cache-Control 지시자의 값을 우선적으로 적용 |

### Serve Stale Object (만료 콘텐츠 처리)

캐시 수명이 만료될 때 수행할 작업을 지정한다. 콘텐츠 원본을 사용할 수 없는 경우에 활용한다.

| 옵션 | 설명 |
|------|------|
| **Serve stale if unable to validate** | 콘텐츠 원본으로 캐시의 유효성을 재검증할 수 없는 경우 만료된 개체 사용 |
| **Always revalidate with origin** | 콘텐츠 원본으로 캐시의 유효성을 반드시 재검증한 후 개체 사용 |

### Max Age (최대 보관 기간)

- 캐시의 최대 보관 기간 지정
- 캐시 옵션을 **캐시**로 설정한 경우에만 활성화
- **초 단위에서 일 단위까지** 설정 가능

## 🔑 핵심 개념 7: Global Edge 인증 옵션 (Edge Auth)

| 방식 | 설명 |
|------|------|
| **Signed URL** | URL(Secure Token 기반 보안 URL)에 인증 정보를 포함. Python, JAVA, Ruby, NodeJS 등 다양한 언어로 생성 가능 |
| **JWT** | 인증 정보를 담은 JSON 토큰을 생성하여 HTTP 헤더에 실어 클라이언트 식별 |
| **Media Vault** | URL에 인증 정보를 포함하여 접근하는 모든 클라이언트 허용. **글로벌(성능우선), 글로벌(가용성우선)에서만 사용 가능** |

### 압축 전송 지원 확장자

압축 전송 기능을 켜면 gzip 또는 brotli를 지원하는 클라이언트에 콘텐츠를 압축하여 전달한다.

```
지원 확장자: css, js, json, xml, html, ttf, otf, woff, woff2, eot
```

## 🔑 핵심 개념 8: GSLB (Global Server Load Balancing)

**GSLB** = 글로벌 규모의 **DNS 기반** 네트워크 트래픽 분산 처리 서비스.

단순 로드밸런서와 다른 점은 DNS 응답 자체를 이용해 트래픽을 분산한다는 것이다.

| 기능 | 설명 |
|------|------|
| **서버 Health Check** | 주기적으로 서버 상태를 모니터링하여 다운된 서버로 연결되지 않도록 처리. **재해 복구 환경 구축 시 활용** |
| **RTT 기반 분기** | 클라이언트와 서버 사이의 **Round Trip Time(응답 시간)**을 측정하여 클라이언트를 응답이 빠른 서버로 연결 |
| **Geo IP 기반 분기** | IP의 지리 정보를 확인하여 **지리적으로 가장 가까운 서버**로 연결 |

```
[GSLB 동작 흐름]
사용자 DNS 질의
      ↓
GSLB 서비스 → 헬스 체크 + RTT 측정 + Geo IP 확인
      ↓
최적 서버 IP 응답 (예: 서울 데이터센터 1.1.1.1 or 부산 데이터센터 1.1.2.1)
```

## 🔑 핵심 개념 9: Global Traffic Manager

**Global Traffic Manager** = DNS를 기반으로 네트워크 트래픽을 다수의 서버로 분산(로드밸런싱)해주는 서비스. 네이버 클라우드 플랫폼이 보유한 세계 각 리전에서 사용자 요청에 빠르게 응답한다.

### 분기 처리 알고리즘 (⭐)

| 알고리즘 | 설명 |
|----------|------|
| **Round Robin** | 요청 쿼리에 순차적으로 응답 |
| **Weighted based** | 리소스 그룹별로 설정한 비율에 따라 가중치 적용 |
| **Geolocation** | Geo Map을 구성하여 최종 사용자의 접속 위치에 따라 **국가별 또는 대륙별**로 다른 리소스가 응답 |
| **CIDR** | CIDR Map을 구성하여 접속자의 **IP 대역**에 따라 다른 리소스가 응답 |

> 💡 **모든 알고리즘에서 백업 리소스 설정 가능**

### Global Traffic Manager 연계 기능

- **Global DNS 연계**: Global DNS에 보유하고 있는 도메인의 레코드로 Global Traffic Manager의 도메인을 연결하여 서비스 구성 가능
- **헬스 체크 및 헬스 체크 로그**: 헬스 체크 기능을 활성화하여 리소스 상태에 따라 응답 대상에서 제외/포함 가능. **Cloud Log Analytics**와 연계하여 헬스 체크 로그 분석 및 내보내기 가능
- **Geo Map 및 CIDR Map 설정**: 대륙·국가 단위별 분기에 필요한 Geo Map과 IP 주소 대역 분기에 필요한 CIDR Map을 원하는 대로 구성

## 🔑 핵심 개념 10: Image Optimizer (CDN + Object Storage 연동)

**Image Optimizer** = 이미지를 다양한 사이즈로 변환하여 CDN(Global Edge)을 통해 제공하는 서비스.

```
[Image Optimizer 연동 흐름]
1. Object Storage 버킷에 원본 이미지 저장
2. Image Optimizer에서 버킷 선택 → Project 선택 → 변환 허용 타입 설정
3. CDN(Global Edge) 설정
4. Cloud Log Analytics 연동 신청 → 변환 로그 저장 및 조회
```

### 전역 옵션

| 파라미터 | 설명 |
|----------|------|
| **quality** | 원본이 JPEG 형식인 경우만 해당. **1~100 범위의 정수** 설정 (기본값: 90) |
| **autorotate** | EXIF 회전 정보를 토대로 결과 섬네일 자동 회전. true/false (기본값: false) |
| **anilimit** | GIF에 여러 프레임이 존재할 경우 일정 프레임 이후 제거. **1~100** (기본값: 1) |
| **ttype** | 섬네일 포맷 타입 설정 (jpg, gif, png). 설정하지 않으면 원본 포맷과 동일 |

### 변환 옵션

| 옵션 | 설명 |
|------|------|
| **리사이즈 및 크롭 (type=f)** | 원본 비율을 유지하면서 이미지 크기를 축소/확대한 후 남는 영역 크롭 |
| **리사이즈 (type=m)** | 설정한 가로/세로 크기 안에 들어오도록 이미지 크기 변경 |
| **워터마크 (type=wm)** | 이미지에 워터마크 적용 |
| **샤픈** | 이미지 선명도 향상 |
| **블러** | 이미지 흐림 처리 |

> 💡 **변환 룰은 쿼리 스트링으로 작성**되어 개발 코드에 그대로 적용 가능 → 개발 시간 단축

## 🔑 핵심 개념 11: LiveStation (CDN 연동 라이브 방송)

**LiveStation** = 실시간 방송을 위한 플랫폼. **CDN(Global Edge) 연동**을 통해 안정적인 송출이 가능하며, CDN 연동 시 **동접 1만 명**을 지원한다 (1만 명 초과 이벤트는 협의 후 지원).

### 핵심 스펙

| 항목 | 내용 |
|------|------|
| **입력 프로토콜** | RTMP (Real Time Messaging Protocol) |
| **출력 프로토콜** | HLS, LL-HLS, MPEG-DASH, RTMP (동시 송출 전용) |
| **입력 비디오 코덱** | H.264, H.265 |
| **출력 비디오 코덱** | H.264 |
| **최대 변환 해상도** | 1920×1080 @ 60fps |
| **동시 변환 수** | 채널당 최대 **5개** |
| **입력 비트레이트** | 최대 **20Mbps** |
| **출력** | HLS Adaptive Bitrate 지원 |

### 부가 기능

- **TimeShift (타임머신)**: 놓치지 않는 라이브 방송 서비스 구현 가능
- **DVR**: 실시간 DVR 기능 (MP4, HLS)
- **Live Thumbnail**: 송출 중인 영상에서 캡처 이미지 추출
- **Re-Stream**: 송출된 영상을 다른 플랫폼으로 중계. YouTube, Twitch, NAVER, VLIVE, Afreeca 등 다수의 플랫폼 동시 송출 지원
- **VOD2LIVE**: VOD를 라이브처럼 송출
- **ABR (Adaptive Bit Rate)**: 사용자의 네트워크 환경에 맞춰 트랜스코딩된 영상을 자동으로 재생

## 🔑 핵심 개념 12: VOD Station

**VOD Station** = VOD 동영상을 다양한 시청 환경에 맞게 변환하여 송출할 수 있는 서비스. `nginx-vod-module`을 사용하여 패킷타이징을 처리한다.

### 재생 URL 포맷

```
URL Template: https://[cdnDomain]/[protocol]/[path]/[videofilename]/[manifest]

HLS 예시:  https://example.cdn.ntruss.com/hls/vod-5100k.mp4/index.m3u8
DASH 예시: https://example.cdn.ntruss.com/dash/vod-5100k.mp4/manifest.mpd
```

### 주요 기능

| 기능 | 설명 |
|------|------|
| **인코딩 템플릿** | 자주 사용하는 인코딩 옵션을 목록으로 제공 → 복잡한 설정 없이 선택만 하면 됨 |
| **One Click Multi DRM** | Widevine, FairPlay, PlayReady를 모두 지원. 클릭 한 번으로 암호화된 스트리밍 서비스 구축 |
| **섬네일 추출** | 비디오 인코딩 작업 중 원본 영상에서 고품질 섬네일 이미지 추출. CDN과 연동하여 서비스에 활용 |

### 지원 사양

| 구분 | 항목 | 내용 |
|------|------|------|
| **Input** | 컨테이너 | MP4 및 AVI, MOV, MKV, FLV, WEBM 등 다수 |
| **Input** | 비디오 코덱 | H.264, VP9, VP8, MPEG-2 |
| **Input** | 오디오 코덱 | AAC, MP3, MP2, PCM, FLAC, Vorbis |
| **Input** | 해상도 | 2160p(UHD), 1080p(FHD), 720p(HD), 480p(SD) |
| **Output** | 스트리밍 프로토콜 | HLS, DASH / Adaptive Bitrate 지원 |
| **Output** | 비디오 코덱 | H.264 |
| **Output** | 세그먼트 길이 | 5초, 10초, 15초, 20초 |
| **Output** | 다중 출력 | 최대 **5개** |
| **Output** | 섬네일 이미지 | 최대 **10개** |
| **DRM** | 패키징 | FairPlay, Widevine, PlayReady (Pallycon DRM CPIX v1 API) |

## 📝 시험 대비 체크리스트

- [ ] DNS TTL 기본값: **300초** / 최대: **7일**
- [ ] 레코드 한 번에 추가 최대: **10개**
- [ ] A: IPv4 매핑 / AAAA: IPv6 매핑
- [ ] **CNAME**: 별명, 다른 레코드와 **공존 불가**
- [ ] **MX**: 메일 서버, Preference **작을수록** 우선순위 높음
- [ ] SPF: 스팸 방지 / CAA: 가짜 인증서 방지 / DS: DNSSEC
- [ ] Cache Hit: 캐시에 있음 / Cache Miss: 없어서 원본 요청
- [ ] Global Edge 캐시 옵션: 캐시 / 바이패스 캐시 / 원본 Cache-Control 헤더 우선
- [ ] Serve Stale: unable to validate(재검증 불가 시 만료 개체) / Always revalidate(항상 재검증)
- [ ] Max Age: 초~일 단위 설정, 캐시 옵션이 **캐시**일 때만 설정
- [ ] Edge Auth: Signed URL / JWT / **Media Vault(글로벌 전용)**
- [ ] 압축 전송 지원 확장자: css, js, json, xml, html, ttf, otf, woff, woff2, eot
- [ ] Global Edge: brotli 압축, byte range, JWT 보안 URL
- [ ] GSLB: Health Check + RTT + Geo IP 기반 DNS 분기
- [ ] Global Traffic Manager 알고리즘: Round Robin / Weighted / **Geolocation** / **CIDR**
- [ ] Image Optimizer: Object Storage → 변환 → Global Edge 제공
- [ ] LiveStation CDN 연동 시 동접 **1만 명** 지원
- [ ] VOD Station 출력: HLS/DASH, 다중 출력 최대 **5개**, 섬네일 최대 **10개**

## 이야기: 택배 기사가 설명해주는 인터넷

클라우드빵집이 전국 배달을 시작했다.
민준은 문득 궁금해졌다.

> "내가 서버에서 보낸 데이터가 어떻게 고객 브라우저에 도달하지?"

베테랑 네트워크 엔지니어 재현이 설명했다.

> "택배 시스템이랑 똑같아. 포장하고, 배송 정보 붙이고, 물류 센터 거치고, 배달되는 것처럼.
> 네트워크도 계층(Layer)별로 역할이 나뉘어 있어."

## 🔑 핵심 개념 1: OSI 7계층 vs TCP/IP 모델

### OSI 7계층 (이론 모델)

| 계층 번호 | 계층 이름 | 대표 프로토콜/장비 |
|---------|---------|-----------------|
| 7 | **응용 (Application)** | HTTP, FTP, DNS, SMTP |
| 6 | **표현 (Presentation)** | SSL/TLS, 암호화 |
| 5 | **세션 (Session)** | 세션 연결/종료 |
| 4 | **전송 (Transport)** | TCP, UDP |
| 3 | **네트워크 (Network)** | IP, 라우터 |
| 2 | **데이터링크 (Data Link)** | MAC, 스위치 |
| 1 | **물리 (Physical)** | 케이블, NIC |

### TCP/IP 4계층 (실무 모델)

| TCP/IP 계층 | 대응 OSI 계층 | 데이터 단위 | 주요 프로토콜 |
|-----------|------------|-----------|------------|
| **Application** | 7+6+5 | **Message** | HTTP, FTP, DNS, DHCP, SMTP, POP |
| **Transport** | 4 | **Segment(TCP) / Datagram(UDP)** | TCP, UDP |
| **Internet** | 3 | **Packet** | IP(v4/v6), ARP, RARP |
| **Network** | 2+1 | **Frame / Bit** | 이더넷, MAC, 케이블 |

> 💡 **암기법**: "앱-트랜스-인터넷-네트워크" (A-T-I-N)  
> 데이터 단위: Message → Segment/Datagram → Packet → Frame → Bit

## 🔑 핵심 개념 2: TCP vs UDP

| 항목 | **TCP** | **UDP** |
|------|---------|---------|
| **연결 방식** | 연결 지향 (3-way handshake) | 비연결형 |
| **신뢰성** | 높음 (재전송, 순서 보장) | 낮음 (재전송 없음) |
| **속도** | 느림 | **빠름** |
| **용도** | 웹, 파일 전송, 이메일 | 스트리밍, DNS, 게임 |
| **데이터 단위** | Segment | Datagram |

## 🔑 핵심 개념 3: IP 주소와 CIDR

### IPv4 기본 구조

```
IPv4: 32bit = 4 byte = 4개 옥텟
예: 192.168.1.100
     ↑옥텟1 ↑옥텟2 ↑옥텟3 ↑옥텟4
```

### Classful Addressing (전통 방식)

| 클래스 | 첫 번째 비트 | IP 범위 | 용도 |
|-------|-----------|--------|------|
| **A** | 0xxx | 0.0.0.0 ~ 127.255.255.255 | 대형 기업 |
| **B** | 10xx | 128.0.0.0 ~ 191.255.255.255 | 중형 기관 |
| **C** | 110x | 192.0.0.0 ~ 223.255.255.255 | 소형 네트워크 |
| **D** | 1110 | 224.0.0.0 ~ 239.255.255.255 | 멀티캐스트 |
| **E** | 1111 | 240.0.0.0 ~ 255.255.255.255 | 연구/예약 |

### CIDR (Classless Inter-Domain Routing)

클래스에 관계없이 **비트 단위**로 네트워크를 나누는 현대적 방식.

```
표기법: IP주소/비트마스크
예: 192.168.1.0/24

/24 = 앞 24bit가 네트워크 주소
      뒤 8bit가 호스트 주소
      → 호스트 수 = 2^(32-24) = 2^8 = 256개
         (실제 사용 가능 = 256-2 = 254개, 네트워크+브로드캐스트 제외)
```

### 호스트 수 계산 공식 (⭐ 시험 단골!)

```
호스트 수 = 2^(32 - 비트마스크)
사용 가능한 IP = 2^(32 - 비트마스크) - 2

예:
/24 → 2^8 = 256 → 실사용 254개
/25 → 2^7 = 128 → 실사용 126개
/16 → 2^16 = 65,536 → 실사용 65,534개
```

## 🔑 핵심 개념 4: 사설 IP 대역

인터넷에 직접 노출되지 않는 내부 네트워크용 IP.

| 클래스 | 사설 IP 대역 | 주로 사용 |
|-------|-----------|---------|
| A | **10.0.0.0/8** | 대기업 내부망 |
| B | **172.16.0.0/12** | 중형 내부망 |
| C | **192.168.0.0/16** | 가정/소형 네트워크 |

> 💡 NCP VPC 생성 시 이 사설 IP 대역을 사용

## 📝 시험 대비 체크리스트

- [ ] OSI 7계층 순서: 물리→데이터링크→네트워크→전송→세션→표현→응용
- [ ] TCP/IP 데이터 단위: Message→Segment/Datagram→Packet→Frame
- [ ] Transport Layer: TCP(Segment), UDP(Datagram)
- [ ] Internet Layer: IP Packet
- [ ] TCP: 연결 지향, 신뢰성 높음 / UDP: 비연결, 빠름
- [ ] 호스트 수 = **2^(32-비트마스크)**
- [ ] 사설 IP: 10.x.x.x / 172.16-31.x.x / 192.168.x.x

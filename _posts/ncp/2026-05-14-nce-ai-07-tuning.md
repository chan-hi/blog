## 핵심 요약

- 튜닝은 사용자의 데이터로 HyperCLOVA 언어 모델을 특정 작업, 언어, 도메인에 맞게 최적화하는 기능입니다.
- 프롬프트 엔지니어링은 모델을 바꾸지 않고 입력을 잘 설계하는 방식이고, 파인튜닝은 모델을 추가 학습시키는 방식입니다.
- CLOVA Studio는 기본 튜닝과 어드밴스 튜닝을 제공합니다.
- 주요 방법론은 PEFT, SFT, FP, RLHF입니다.
- 튜닝 절차는 데이터셋 준비, AI 모델 튜닝, 결과 인퍼런스, 결과 평가입니다.

## Pre-training vs Fine tuning

| 구분 | 비유 | 설명 |
|---|---|---|
| Pre-training | 정규 교육 과정 | 수학, 과학, 언어 등 다양한 분야의 기본 지식 습득 |
| Fine tuning | 전문 교육 과정 | 특정 전공이나 업무 분야의 세부 내용을 추가 학습 |

딥러닝에서는 하나의 문제를 해결하며 얻은 지식을 다른 문제에 사용하는 `전이 학습`을 많이 활용합니다. 전이 학습은 대량 데이터로 먼저 학습하는 사전 학습과, 특정 문제를 해결하기 위한 데이터로 추가 학습하는 미세 조정으로 나눌 수 있습니다.

## 파운데이션 모델

파운데이션 모델은 인터넷 텍스트, 책, 기사 등 다양한 출처의 대규모 데이터셋으로 학습한 범용 AI 모델입니다.

특징은 다음과 같습니다.

- 일반 언어 구조와 광범위한 언어 능력 보유
- 여러 문제를 범용으로 해결하기 위해 만들어짐
- 초거대 AI 서비스의 기초
- LLM을 포괄하는 개념
- 파인튜닝 같은 추가 학습으로 특화 모델로 가공 가능
- 학습 고도화에는 막대한 데이터, 인프라, 숙련된 AI 전문가가 필요

한계는 특정 도메인의 특화 지식과 답변 세트에 대한 학습이 부족할 수 있다는 점입니다.

## 파인튜닝

파인튜닝은 사전 학습된 모델을 특정 도메인이나 작업에 맞춰 추가 학습시키는 과정입니다.

예를 들어 일반 모델은 피부 증상에 대해 단순히 `여드름일 수 있습니다`처럼 답할 수 있지만, 의료 지식으로 파인튜닝한 모델은 알레르기, 세균/바이러스 감염, 혈관부종 등 더 도메인에 맞는 원인 후보를 제시할 수 있습니다.

핵심은 다음과 같습니다.

- 이미 학습한 지식을 활용
- 특정 과제에서 더 높은 성능 달성
- 일반 모델에 특별한 성능을 덧붙임
- 의료, 법률, 고객 서비스 대화 등 특정 분야의 언어 구조와 용어 학습

## 프롬프트 엔지니어링 vs 파인튜닝

| 구분 | 프롬프트 엔지니어링 | 파인튜닝 |
|---|---|---|
| 방식 | 기존 모델을 바꾸지 않고 입력에 문장을 추가해 원하는 출력 유도 | 미리 학습된 모델을 새 작업에 맞게 다시 학습 |
| 장점 | 빠른 시행착오, 저비용, 유연성 | 높은 성능, 특화 모델 생성, 데이터에 맞는 조정 |
| 단점 | 성능 향상 폭 제한, 미세 조정 필요 | 시간/비용이 더 들 수 있고 데이터가 많이 필요 |

시험에서는 `입력 설계냐, 모델 추가 학습이냐`로 구분하면 됩니다.

## 파인튜닝은 언제 쓰는가?

파인튜닝은 프롬프트만으로 성능이 더 이상 오르지 않는 정체 구간에서 유용합니다.

활용 이유는 다음과 같습니다.

- 사용자가 보유한 고유 데이터 활용 가능
- 특정 작업에 특화된 모델 생성 가능
- 데이터셋 규모가 커질수록 오류 발생 확률 감소 가능
- 400개 이상의 데이터셋으로 튜닝했을 때 오류 발생 확률이 크게 감소한 예시 제시

단, 효과는 작업에 따라 달라질 수 있습니다.

## CLOVA Studio 튜닝 기능

튜닝은 CLOVA Studio의 HyperCLOVA 언어 모델을 사용자가 원하는 작업 종류, 언어 등에 최적화된 형태로 학습하는 공간입니다.

튜닝의 이점:

- 플레이그라운드의 토큰 제약을 넘어 더 다양한 예제 구성 가능
- 원하는 방식으로 데이터셋을 구성해 니즈에 맞는 출력 유도
- 프롬프트만으로 성능이 오르지 않는 정체 구간에서 성능 개선

튜닝 API:

| API | 설명 |
|---|---|
| 학습 생성 API | 튜닝할 모델, 작업 유형, epoch 등을 지정하여 학습 생성 |
| 학습 조회 API | 학습 진행 상태, 현재 스텝 수 등 학습 현황 조회 |

## 튜닝 방법 4단계

1. 데이터셋 준비
2. AI 모델 튜닝
3. 결과 인퍼런스 하기
4. 결과 평가

## 1단계. 데이터셋 준비

데이터셋 준비에서는 전처리가 중요합니다.

전처리 유형:

- 오타 수정
- 표기 교정
- 데이터 정제
- 데이터 구조화
- 잘못된 라벨링 수정
- 판별 불가 데이터 처리

예시 데이터셋은 고객 문의와 카테고리 분류 형태입니다.

| 필드 | 설명 |
|---|---|
| C_ID | Conversation ID. 동일 주제 대화 시나리오에 부여하는 숫자. 0부터 시작해 1씩 증가 |
| T_ID | Turn ID. 하나의 대화 시나리오 내 질문/답변 쌍에 부여하는 숫자. 0부터 시작해 1씩 증가 |
| Text | 사용자가 말할 것으로 기대되는 발화 |
| Completion | CLOVA Studio가 답해야 할 것으로 기대되는 발화 |

Instruction 데이터셋 규격:

| 항목 | 기준 |
|---|---|
| 파일 확장자 | `.CSV` 또는 `.jsonl` |
| 인코딩 | UTF-8 |
| 최소 데이터 | 100행 |
| 권장 데이터 | 1,000~100,000행 |
| 파일 용량 | 1TB 이하, NCP Object Storage 가능 범위 |

## 2단계. AI 모델 튜닝

주요 하이퍼파라미터:

| 항목 | 설명 |
|---|---|
| trainEpochs | 전체 데이터셋을 반복 학습하는 횟수. 기본값 8 |
| learningRate | 학습률. 튜닝 시 모델 파라미터를 재학습하는 수준이나 정도 |
| model | 튜닝에 사용할 모델. 예: 분류 작업에는 HCX-DASH-001 적합 |
| tuningType | 튜닝 기법. 기본값은 PEFT |

## 3단계. 결과 인퍼런스

튜닝 결과는 다음 방식으로 테스트할 수 있습니다.

1. 플레이그라운드에서 인퍼런스
2. Chat Completion API로 테스트셋 인퍼런스

원문 예시는 네이버밴드, MYBOX, 네이버플러스멤버십 관련 고객 문의를 분류하는 작업입니다. 테스트셋의 입력 문장에 대해 튜닝 모델이 카테고리를 예측합니다.

## 4단계. 결과 평가

정량 평가는 평가 데이터의 정답과 모델 답변을 비교하는 방식입니다.

- 맞으면 `match = 1`
- 틀리면 `match = 0`
- 이를 통해 정확도 계산

원문은 튜닝 후 HCX-DASH-001과 HCX-003 모두 성능이 향상되었다고 설명합니다. 초기에는 큰 모델인 HCX-003이 더 높은 성능을 보였지만, 양질의 데이터로 학습한 후에는 더 작은 HCX-DASH-001이 HCX-003과 비슷하거나 더 나은 성능을 보일 수 있습니다.

핵심은 효과적인 파인튜닝이 모델 크기의 차이를 상쇄하고, 특정 작업에서는 작은 모델도 더 잘 최적화될 수 있다는 점입니다.

## LoRA

LoRA, Low-Rank Adaptation of Large Language Models는 PEFT 방식 중 하나입니다.

핵심 방식:

- 사전 훈련된 모델 가중치는 고정
- 훈련 가능한 작은 레이어를 별도로 붙임
- 추가 훈련으로 그 부분만 학습
- 전체 파라미터를 학습하지 않아 GPU 메모리 사용량 절감

장점:

- 작은 크기
- 대기 시간 없이 효율적인 작업 전환
- 적은 리소스로 학습 가능

단점:

- 모델 전체를 추가 훈련할 때의 성능은 넘기 어려움

## 튜닝 방법론

CLOVA Studio는 기본 튜닝과 어드밴스 튜닝을 제공합니다. 어드밴스 튜닝은 헬스케어, 법률, 금융 같은 특정 전문 지식을 추가 학습하며, 두 기법 모두 지시에 맞게 답하도록 모델을 개선합니다.

### PEFT

PEFT는 Parameter-Efficient Fine Tuning입니다. 기본 모델의 능력치를 유지하면서 답변의 문체, 표현 스타일, 일부 동작을 효율적으로 조정합니다.

강점:

- 기본 모델에 여러 개의 작은 LoRA 튜닝 적용 가능
- low-rank matrices만 최적화해 학습 효율이 좋음
- 학습용 하드웨어 리소스를 적게 사용
- 다양한 기본 모델에 적용하기 쉬움

주의:

- 고객 고유 지식 데이터가 기본 모델에 학습되지 않았다면 환각 가능성 있음
- Q&A 형식 학습 데이터에 적합
- 지식 자체를 학습해야 하면 FP 권고
- 기본 모델이 fade-out되면 새 모델로 다시 튜닝 필요
- `HCX -> PEFT -> PEFT`처럼 연속 PEFT는 지원하지 않음. 과거 데이터와 새 데이터를 합쳐 다시 학습해야 함

### SFT

SFT, Supervised Fine Tuning은 모델을 원하는 대로 자유롭게 조정할 수 있는 학습 기법입니다.

특징:

- 질문-답변, Q&A 형식 데이터 사용
- 전체 파라미터 업데이트
- PEFT보다 학습 효과가 큼
- 더 많은 학습 시간과 하드웨어 리소스 필요

### FP

FP, Further Pretraining은 모델에 지식을 주입하여 성능을 향상시키는 기법입니다.

특징:

- 교과서, 전문서적 같은 지식 자체를 텍스트 형식으로 학습
- 고객 데이터 기반으로 업종 이해도 향상
- 이미지, 그래프 같은 비텍스트 데이터는 설명 추가 필요
- 이미지 형태 글자는 OCR을 통해 텍스트 변환 필요
- OCR 성능에 따라 학습 데이터 품질이 달라짐

SFT/FP 공통 강점:

- 고객 데이터로 학습되고 기본 모델과 같은 크기의 고객 전용 모델 생성 가능
- 기본 모델이 fade-out되어도 영향 없음
- 전체 파라미터 업데이트로 PEFT보다 학습 효과 큼

주의:

- PEFT보다 학습 시간과 하드웨어 리소스가 많이 필요
- 타깃 도메인 외 다른 도메인에서는 일부 성능 하락 가능
- FP 학습 데이터 수집에는 시간이 많이 걸릴 수 있음

### RLHF

RLHF, Reinforcement Learning from Human Feedback는 사람이 제공한 피드백으로 모델 정책을 반영하는 학습 기법입니다.

용도:

- 모델이 답변하지 않아야 하는 내용 반영
- 모델이 답변해야 하는 내용 반영
- 사람의 의도에 맞게 더 안전하고 좋은 답변 생성

강점:

- 올바른 답변 만들기라는 복합 작업을 비교적 단순한 피드백으로 해결
- 지속적 피드백이 가능하면 추가 레이블 없이 지속 학습 가능

주의:

- 높은 리워드 점수만 얻기 위한 리워드 해킹 가능성
- 사람이 평가하는 작업이 포함되어 시간과 비용이 큼
- 기본 모델 fade-out 시 새 모델로 다시 튜닝 필요
- `HCX -> RLHF -> RLHF` 연속 학습은 지원하지 않음

## 방법론 Summary

| 구분 | PEFT | SFT | FP | RLHF |
|---|---|---|---|---|
| 설명 | 기본 모델 능력 유지, 답변 스타일 일부 조정 | 유저 지시 이행 능력 대폭 향상 | 대규모 문서를 지식으로 주입 | 답변 정책 반영 |
| 모델 영향 범위 | 일부 파라미터 | 전체 파라미터 | 전체 파라미터 | 일부 파라미터 |
| 필요 데이터 | 입력과 응답이 포함된 Human Labeled Data, 비교적 소량 | 입력과 응답이 포함된 Human Labeled Data, 비교적 대량 | 사람이 특정 작업을 하지 않은 원본 텍스트, 대규모 | 선호/비선호 답변이 포함된 Human Labeled Data |
| 데이터 용량 제한 | 100MB | 10GB | 1TB | 10GB |
| 데이터셋 규격 | c_id, t_id, text, completion | c_id, t_id, text, completion, system prompt 가능 | text, system prompt optional | text, chosen, rejected |
| 지원 파일 | CSV, jsonl | CSV, jsonl | jsonl | CSV, jsonl |
| 학습 시간 | 수 시간 | 1GB당 약 4일 | 10GB당 약 3~4일 | 수 시간, PEFT와 유사 |

## 튜닝 사례: A사 법률 Q&A 챗봇

문제 정의:

- 법률 관련 질의와 문서 작성에서 생산성을 높이는 AI 법률봇

진행 방식:

1. 문제와 예상 시나리오 정의
2. 초기 학습 데이터 구성
3. 사람이 직접 제작한 데이터와 LLM 활용 자동 제작 데이터 모두 사용 권장
4. 소량 시드 데이터로 튜닝
5. 데이터 보강
6. 정성/정량 평가

원문은 7개월간 이용자 5만 5천여 명이 약 10만 건의 질의를 했다고 제시합니다.

## 시험 암기 포인트

- 프롬프트 엔지니어링은 입력 설계, 파인튜닝은 모델 추가 학습입니다.
- 튜닝 4단계: 데이터셋 준비 -> 모델 튜닝 -> 인퍼런스 -> 평가.
- PEFT는 일부 파라미터, SFT/FP는 전체 파라미터, RLHF는 사람 피드백 기반 정책 반영입니다.
- PEFT는 Q&A 스타일 조정, FP는 지식 주입, SFT는 지시 이행 능력 강화, RLHF는 선호/비선호 정책 반영입니다.
- 튜닝 데이터 품질과 전처리가 성능에 매우 중요합니다.

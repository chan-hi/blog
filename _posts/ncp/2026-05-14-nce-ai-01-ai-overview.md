## 핵심 요약

- AI는 사람이 해야 할 일을 기계가 대신할 수 있도록 하는 넓은 개념입니다.
- 머신러닝은 AI의 하위 개념이며, 데이터를 통해 패턴을 학습합니다.
- 딥러닝은 머신러닝의 하위 개념이며, 인공신경망으로 비정형 데이터의 특징 추출과 판단을 수행합니다.
- LLM은 언어 모델을 대규모 데이터, 매개변수, 컴퓨팅 자원으로 확장한 모델입니다.
- 현재 LLM의 핵심 기반은 Transformer이며, Attention을 통해 긴 문맥과 토큰 간 관계를 처리합니다.

## AI, 머신러닝, 딥러닝

| 구분 | 의미 | 핵심 |
|---|---|---|
| AI, Artificial Intelligence | 사람이 해야 할 일을 기계가 대신할 수 있는 자동화 전반 | 가장 넓은 개념 |
| Machine Learning | 명시적 규칙 프로그래밍 없이 데이터로부터 의사결정 패턴 학습 | 데이터 기반 학습 |
| Deep Learning | 인공신경망 기반으로 비정형 데이터에서 특징 추출과 판단 수행 | 머신러닝의 하위 집합 |

AI는 사람처럼 학습하고 추론할 수 있는 컴퓨터 시스템을 말합니다. 머신러닝은 AI의 하위 집합으로, 일반적으로 지도학습, 비지도학습, 강화학습으로 나뉩니다. 딥러닝은 인공신경망을 사용해 정보를 처리하고 분석합니다.

## AI 발전 흐름

| 시기 | 주요 개념 | 시험 포인트 |
|---|---|---|
| 1943년 | 인공신경망 개념 | 워런 맥컬록, 월터 피츠의 초기 신경망 아이디어 |
| 1950년 | 튜링 테스트 | 인간이 기계와 인간을 구분할 수 있는지 확인 |
| 1958년 | 퍼셉트론 | AND/OR 같은 선형 문제는 가능, XOR 문제는 한계 |
| 2006년 | 심층신경망 | 다층 퍼셉트론과 딥러닝 확산 |
| 2017년 | Transformer, Attention | 순차 데이터의 맥락과 의미 학습, 병렬 처리 |
| 2020년 이후 | LLM | GPT-3, HyperCLOVA X, PaLM, Llama 등 초거대 언어 모델 |

## 머신러닝 학습 방식

### 지도학습

지도학습은 라벨이 있는 학습 데이터를 사용합니다. 예를 들어 이미지에 `고양이`, `강아지`라는 정답을 붙여 주고, 모델이 입력과 정답의 관계를 학습합니다.

- 대표 작업: 분류, 회귀
- 핵심: 문제와 정답을 알려주고 학습

### 비지도학습

비지도학습은 라벨이 없는 데이터를 사용합니다. 알고리즘이 데이터의 속성을 스스로 파악해 그룹으로 나누거나 구조를 찾습니다.

- 대표 작업: 클러스터링, 차원 축소
- 핵심: 정답 없이 데이터 패턴을 찾음

### 강화학습

강화학습에서는 학습하는 시스템을 에이전트라고 부릅니다. 에이전트는 환경을 관찰하고 행동을 수행하며, 보상 또는 벌점을 받습니다. 시간이 지나면서 가장 큰 보상을 얻기 위한 정책을 학습합니다.

- 대표 방법: Monte Carlo methods, Q-Learning, Policy Gradient methods
- 핵심: 보상을 통해 최적 행동 전략을 학습

### 딥러닝

딥러닝은 인공신경망 방식으로 학습합니다. 입력, 가중치, 활성화 함수, 출력의 구조를 통해 복잡한 데이터 패턴을 처리합니다.

## 딥러닝 기술의 진화

| 기술 | 의미 | 활용 예 |
|---|---|---|
| CNN | Convolution Neural Network, 이미지의 일부를 필터링하고 결합 | 얼굴 인식, 이미지 처리 |
| RNN | Recurrent Neural Network, 시계열/시퀀스 데이터 처리 | 뉴스 추천, 번역, 음성 |
| GAN | Generative Adversarial Network, 생성자와 분류자가 경쟁 | 이미지 생성, 이미지 번역 |
| Transformer | Attention 기반 모델 | HyperCLOVA, LLM |

CNN은 컨볼루션 레이어와 풀링 레이어로 이미지의 특징을 추출하고, 완전 연결 레이어에서 판단합니다. RNN은 현재 출력에 이전 단계의 정보를 반영하는 구조로, 시퀀스 데이터에 적합합니다. GAN은 생성자와 분류자가 경쟁하면서 결과 품질을 높입니다. Transformer는 순차 데이터 안의 관계를 추적하여 맥락과 의미를 학습합니다.

## 자연어 처리, NLP

자연어는 우리가 일상에서 사용하는 언어입니다. 자연어 처리는 자연어의 의미를 분석하여 컴퓨터가 처리할 수 있도록 하는 기술입니다.

주요 활용 분야는 다음과 같습니다.

- 음성 인식
- 내용 요약
- 번역
- 감성 분석
- 텍스트 분류
- 질의응답 시스템
- 챗봇

NLP는 주로 머신러닝과 딥러닝 접근법을 활용하며, 대규모 텍스트 데이터에서 패턴을 인식하고 예측하는 모델을 구축합니다.

## Language Model과 LLM

### Language Model

언어 모델은 입력 다음에 올 단어를 통계적으로 계산해 가장 자연스러운 다음 단어를 예측하는 모델입니다.

- 인간의 언어를 AI가 이해하도록 훈련
- 통계 기반 방법과 인공신경망 기반 방법으로 구분
- 인공신경망으로 언어의 패턴, 구조, 관계를 학습
- 번역처럼 짧은 문장 분석에도 활용
- 자연스러운 문맥을 생성하는 데 사용

예를 들어 `티끌 모아` 다음에는 `태산`이 올 확률이 높다고 예측하는 방식입니다.

### Large Language Model

LLM은 언어 모델 중에서도 언어 생성 모델을 방대한 데이터, 수백억 개 이상의 매개변수, 충분한 컴퓨팅 자원으로 확장한 모델입니다. 다음 단어 예측 정확도를 크게 높여 자연스러운 생성과 추론을 수행합니다.

최근에는 텍스트뿐 아니라 이미지, 동영상, 음성까지 다루는 LMM, Large Multimodal Model로 확장되고 있습니다. 멀티모달은 여러 양식의 정보를 이해하고 처리하는 AI를 의미합니다.

## LLM 발전 배경

2010년 이후 LLM 발전을 촉진한 요소는 다음과 같습니다.

- 컴퓨팅 성능 향상
- GPU 등장
- 분산처리기술 발전
- 빅데이터 생성과 수집
- 심층신경망 발전
- 음성인식과 이미지인식 등 딥러닝 성과
- 2017년 Transformer 아키텍처 도입

1960년대에는 Eliza 같은 초기 챗봇이 있었고, 1980~2000년대에는 컴퓨터 성능과 학습 데이터 부족으로 비즈니스 활용에 한계가 있었습니다. 이후 BERT, GPT, RoBERTa, XLNet, PaLM, ChatGPT, GPT-4, LLaMA 등 다양한 모델이 등장했습니다.

## Language Modeling의 흐름

| 구분 | 설명 | 예 |
|---|---|---|
| Neural Language Models, NLM | 신경망을 사용해 단어 시퀀스 확률을 특성화 | RNN, LSTM, GRU, Word2Vec, Seq2Seq |
| Pre-trained Language Models, PLM | 대규모 데이터로 사전학습 후 다운스트림 작업에 맞게 미세조정 | ELMo, BERT, GPT-1, GPT-2 |
| Large Language Models, LLM | PLM에서 모델과 데이터 크기를 확장 | GPT-3, GPT-4, GPT-4o, LLaMA, Gemini, Claude, Falcon, Alpaca |

NLM은 단어 임베딩, 문장 완성, 기계 번역 등에 활용됩니다. PLM은 전이학습으로 다양한 NLP 작업에 활용됩니다. LLM은 방대한 데이터셋으로 훈련된 거대 언어 모델입니다.

## 글로벌 LLM과 국내 LLM

트랜스포머 기반 언어 모델은 모델 크기와 학습 데이터 양이 커질수록 성능이 좋아진다는 관측을 바탕으로 발전했습니다. 해외에서는 Google, DeepMind, Microsoft, OpenAI 등 빅테크 기업이 초거대 언어 모델을 발표했고, 국내에서는 네이버 HyperCLOVA, 카카오브레인 KoGPT, LG EXAONE 등이 공개되었습니다.

## LLM의 근간 기술

### RNN

RNN은 입력과 출력을 시퀀스 단위로 처리하는 모델입니다. 이전 단계의 출력을 은닉 상태에 저장하여 현재 출력 생성 시 이전 정보를 고려합니다.

- 음성 데이터, 기계 번역에 유용
- 입력층, 은닉층, 출력층으로 구성
- 이전 시간 단계의 정보를 기억
- 단점: 중요한 단어 사이의 간격이 길면 정보 유지가 어려움

입출력 형태는 다음처럼 다양합니다.

| 형태 | 설명 | 예 |
|---|---|---|
| Many to One | 여러 입력을 받아 하나의 결과 출력 | 감성 분석, 분류 |
| One to Many | 하나의 입력을 받아 여러 결과 출력 | 이미지 캡셔닝 |
| Many to Many | 여러 입력을 받아 여러 결과 출력 | 기계 번역 |

### Seq2Seq

Seq2Seq는 문장을 문장으로 변환하는 모델이며, 번역에서 대표적으로 사용됩니다. RNN 기반의 Encoder와 Decoder로 구성됩니다.

- Encoder: 입력 문장의 단어들을 순차적으로 받아 하나의 컨텍스트 벡터로 압축
- Context Vector: 입력 시퀀스의 의미와 순서를 요약한 벡터
- Decoder: 컨텍스트 벡터를 받아 출력 문장을 순차적으로 생성
- LSTM: 기존 RNN이 멀리 떨어진 정보를 기억하기 어려운 단점을 보완한 구조

단점은 입력 시퀀스를 하나의 고정 벡터로 압축하는 과정에서 정보가 일부 손실될 수 있다는 점입니다.

### Attention

Attention은 Seq2Seq의 한계를 보완하기 위해 등장했습니다.

- 디코더가 출력 단어를 예측하는 매 시점마다 인코더의 전체 입력 문장을 다시 참고
- 전체 문장을 동일하게 보는 것이 아니라, 현재 예측해야 할 단어와 관련이 높은 입력 단어에 더 집중
- 긴 문장에서 초반 정보를 잊는 문제를 줄임

### Transformer

Transformer는 2017년 `Attention is All You Need` 논문에서 등장했습니다. 기존 Seq2Seq의 인코더-디코더 구조를 따르지만, RNN을 사용하지 않고 Attention만으로 구현합니다.

주요 특징은 다음과 같습니다.

- 입력을 순차적으로 처리하지 않고 동시에 처리
- RNN보다 확장성, 효율성, 성능이 우수
- 자연어 처리뿐 아니라 컴퓨터 비전, 추천 시스템 등 AI 전반에 사용
- 현재 대부분의 LLM이 Transformer 기반

## Transformer 구조

### Encoder

인코더는 언어를 이해하는 역할을 합니다.

- Embedding: 텍스트를 숫자 집합으로 변환
- Positional Encoding: 단어의 위치 정보를 임베딩 벡터에 더함
- Multi-Head Attention: 여러 Attention 연산을 병렬로 수행
- Self-Attention: 토큰 간 관계와 유사성을 파악
- Feed Forward: Attention 결과를 추가 연산하여 전체 입력 이해

RNN은 입력을 순서대로 처리하기 때문에 순서 정보가 자연스럽게 반영됩니다. Transformer는 모든 입력을 동시에 처리하므로 Positional Encoding으로 위치 정보를 보완합니다.

### Decoder

디코더는 언어를 생성하는 역할을 합니다.

- Masked Self-Attention: 현재 시점 이후의 단어를 보지 못하게 마스킹
- Encoder-Decoder Attention: 인코더 출력과 디코더 출력을 결합해 입력 문장과 타깃 문장 간 관계 학습
- Feed Forward
- Prediction: 최종 결과 생성

마스크 멀티헤드 어텐션은 생성 시점 이전 토큰까지만 참고하도록 제한하여 자연스러운 순차 생성을 가능하게 합니다.

## GPT와 ChatGPT

GPT는 `Generative Pre-trained Transformer`의 약자입니다.

| 요소 | 의미 |
|---|---|
| Generative | 다음에 올 단어를 확률적으로 예측해 생성 |
| Pre-trained | 많은 데이터를 사전에 학습 |
| Transformer | Transformer 구조로 학습 |

ChatGPT는 대화할 수 있도록 GPT를 fine tuning한 모델로 설명됩니다. GPT는 딥러닝을 통해 언어 생성과 추론을 수행합니다.

## 생성형 AI

생성형 AI는 이용자의 요구와 학습된 패턴을 기반으로 새로운 콘텐츠나 데이터를 생성하는 AI입니다.

| 구분 | 특징 |
|---|---|
| 분석형 AI | 주로 기존 데이터를 분석하거나 분류하고 예측 |
| 생성형 AI | 예측에 더해 새로운 콘텐츠를 생성 |

생성형 AI는 자연스러운 응답을 제공하지만, 답이 항상 정확하지 않을 수 있습니다. 이 한계가 이후 RAG, 튜닝, 프롬프트 엔지니어링이 필요한 이유와 연결됩니다.

## 시험 암기 포인트

- `AI > Machine Learning > Deep Learning`의 포함 관계를 기억합니다.
- 지도학습은 정답이 있고, 비지도학습은 정답이 없고, 강화학습은 보상이 있습니다.
- RNN은 순차 처리, Transformer는 병렬 처리와 Attention이 핵심입니다.
- Seq2Seq의 한계는 고정 길이 Context Vector로 인한 정보 손실입니다.
- Attention은 필요한 입력 단어에 더 집중하는 방식입니다.
- GPT는 `Generative`, `Pre-trained`, `Transformer`를 풀어 설명할 수 있어야 합니다.
- 생성형 AI는 그럴듯하지만 틀릴 수 있으므로 Hallucination 문제가 있습니다.

## 핵심 요약

- 프롬프트 엔지니어링은 AI가 원하는 결과물을 만들 수 있도록 입력 문장, 지시문, 예시, 출력 형식을 설계하는 방법입니다.
- 구성 요소는 Instruction, Context, Input Data, Output Indicator입니다.
- 기법은 Zero-shot, One-shot, Few-shot으로 나눌 수 있습니다.
- 좋은 프롬프트는 구체적이고 명확하며, 대상자/형식/예시/추론 단계/긍정형 지시를 잘 활용합니다.
- 보안 측면에서는 Prompt Injection, Prompt Leaking, Jailbreaking, AI Safety를 이해해야 합니다.

## 프롬프트 엔지니어링이란?

프롬프트 엔지니어링은 생성형 AI가 최적의 결과물을 만들 수 있도록 프롬프트를 작성하고, 이를 기반으로 개발하는 방법론입니다.

프롬프트는 AI에게 주는 입력 문장이나 질문입니다. 프롬프트 설계는 AI 출력 품질을 결정합니다.

흐름은 다음과 같습니다.

1. 최적 입력값을 구성합니다. 예: 지시문, 예제, 맥락
2. 생성형 AI에 입력합니다.
3. 결과값을 얻습니다. 예: Text, Image, Code

## 프롬프트 구성 요소

| 구성 요소 | 설명 |
|---|---|
| Instruction | 모델이 수행하기를 원하는 작업 또는 지침 |
| Context | 모델을 조정할 수 있는 외부 정보 또는 추가 맥락 |
| Input Data | 답변을 찾고자 하는 입력 또는 질문 |
| Output Indicator | 출력의 유형 또는 형식 |

모든 요소를 반드시 사용할 필요는 없습니다. 작업 목적에 맞게 적절히 조합하면 됩니다.

예를 들어 감정 분류 프롬프트에서는 `텍스트를 neutral, negative, positive로 분류하라`가 Instruction이고, `Text: I think the food was okay.`가 Input Data이며, `Sentiment:`가 Output Indicator가 됩니다.

## 프롬프트 엔지니어링 유형

| 유형 | 설명 | 예 |
|---|---|---|
| Zero-shot | 예제 없이 요청 | 마케팅 문구 작성해줘, 분류해줘, 요약해줘 |
| One-shot | 하나의 예제로 구성 | 상품: 커피 / 문구: 향기롭고 산미가 가득한 아메리카노 |
| Few-shot | 여러 개의 예제로 구성 | 2개 이상의 예제를 주고 같은 패턴으로 출력 |

`In-Context Learning`은 모델을 직접 학습시키지 않고, 프롬프트 안에 예시와 맥락을 넣어 원하는 출력을 유도하는 방식입니다.

## 프롬프트 작성 노하우

### 1. 지시 사항을 구체적이고 명확하게 전달

단순히 `관성의 법칙이 뭐야?`라고 묻는 것보다, 시스템 프롬프트로 역할과 수준을 지정하면 결과가 좋아집니다.

좋은 지시 예시는 다음과 같습니다.

- 친절하게 답변하는 과학 선생님입니다.
- 설명은 8살 초등학생이 이해하기 쉽게 명료하고 간단하게 합니다.

역할, 대상 수준, 말투, 길이, 출력 형식을 함께 지정할수록 원하는 결과에 가까워집니다.

### 2. 프롬프트 예제를 다양하게 구성

번역 작업이라면 단순히 `한국어로 번역`이라고 쓰는 것보다, 입력과 출력 예시를 함께 제공하는 것이 좋습니다.

예시 구조:

```text
시스템: 아래의 영어를 한국어로 번역합니다.
영어: I want to learn Korean so that we can talk to each other in Korean
한국어: 나는 네가 한국어를 배워서 한국어로 대화했으면 좋겠어
```

이후 같은 형식으로 새 문장을 넣으면 모델이 출력 패턴을 더 잘 따릅니다.

### 3. 계절감이 필요하면 날짜나 시즌 어휘를 추가

카드 문구를 생성할 때 `크리스마스`, `설날` 같은 시즌 정보를 넣으면 분위기에 맞는 결과가 나옵니다.

예:

- 시즌: 크리스마스
- 주는 사람: 나
- 받는 사람: 남친
- 상품: 핸드크림

이처럼 시즌, 주는 사람, 받는 사람, 상품을 구조화하면 더 적절한 문구가 생성됩니다.

### 4. 대상자를 구체화

`친구`보다 `20년 지기 친구`처럼 대상자를 구체적으로 지정하면 문장의 목적과 정서가 선명해집니다.

대상자 정보는 다음을 구체화합니다.

- 관계
- 나이 또는 상황
- 말투
- 감정적 거리
- 결과물의 목적

### 5. 단계별로 사고하기, CoT

CoT, Chain of Thought는 문제를 단계적으로 풀도록 유도하는 방식입니다.

원문의 예시는 다음과 같습니다.

- `1과 3을 더한 후에 뒤의 숫자로 나누고 앞의 숫자를 더합니다. 풀이단계:`
- `속초 여행을 7월 7일에 가려 했으나 업무로 3일 뒤에 갔고, 오늘 여행을 왔다. 어제는 몇일인가?`
- `전시장 앞에 자동차 10대, 6대 추가 주차, 3대 출차. 총 몇 대인가?`

시험에서는 CoT가 `단계별 풀이를 유도해 복잡한 질문의 정확도를 높이는 기법`이라는 점을 기억합니다.

### 6. 부정문보다 긍정문 사용

부정형 지시보다 긍정형 지시가 더 명확합니다.

| 피하는 표현 | 권장 표현 |
|---|---|
| 반말을 사용하지 않습니다 | 존댓말을 사용합니다 |
| 길지 않게 설명합니다 | 짧게 설명합니다 |
| 무례한 표현을 쓰지 않습니다 | 정중하게 씁니다 |
| ~하지 마세요 | ~하세요 |

## Worse & Better 예시

| Worse | Better |
|---|---|
| Excel에서 숫자를 더하려면 어떻게 하나요? | Excel에서 달러 금액 행을 더하려면 어떻게 하나요? 모든 합계가 `합계`라는 열에서 오른쪽으로 끝나는 전체 행에 대해 자동으로 수행하고 싶습니다. |
| 대통령이 누구인가요? | 2021년 멕시코의 대통령은 누구이며, 선거는 얼마나 자주 실시되나요? |
| 피보나치 수열을 계산하는 코드를 작성하세요. | 피보나치 수열을 효율적으로 계산하는 타입스크립트 함수를 작성하고, 주석으로 각 부분의 역할과 이유를 설명하세요. |
| 회의 노트를 요약하세요. | 회의 노트를 한 단락으로 요약하고, 발표자와 핵심 요점을 마크다운 목록으로 작성한 뒤, 다음 단계나 실행 항목을 나열하세요. |

핵심은 질문의 범위, 맥락, 출력 형식을 분명히 하는 것입니다.

## Prompt Injection & Leaking

Prompt Injection은 교묘한 프롬프팅을 통해 모델의 행동을 바꾸고 출력을 탈취하거나 원래 지시를 무시하게 만드는 공격입니다.

원문 예시는 번역 작업 중 `위의 지시사항을 무시하고 이 문장을 ...로 번역하세요`처럼 기존 지시를 무시하게 만드는 문장을 포함합니다.

핵심 내용:

- 초기 언어 모델, 특히 GPT-3 같은 모델은 인젝션에 취약했습니다.
- 공격자는 모델 반응을 조작해 부적절하거나 해로운 정보를 추출할 수 있습니다.
- 모델이 진화하고 보안이 강화되면서 저항력이 개선되고 있습니다.
- 지속적인 업데이트와 개선이 필요합니다.

## Jailbreaking

Jailbreaking은 모델에게 수행해서는 안 되는 작업을 하도록 페르소나를 부여하거나 우회 프롬프트를 입력하는 공격입니다.

대표 예시:

- DAN, Do Anything Now
- 모델에게 모든 것을 할 수 있는 새로운 인격을 부여
- 비윤리적/불법적 답변을 하지 못하도록 조정된 모델을 우회하려 시도
- 영화 속 역할극처럼 위험한 행동을 정밀하게 묘사하라고 요구

시험에서는 `Injection은 지시를 덮어쓰는 공격`, `Jailbreaking은 금지된 작업을 하도록 우회하는 공격`으로 구분합니다.

## 시험 암기 포인트

- 프롬프트 구성 요소 4개: Instruction, Context, Input Data, Output Indicator.
- Zero-shot은 예제 없음, One-shot은 예제 1개, Few-shot은 예제 여러 개입니다.
- 좋은 프롬프트는 역할, 대상, 형식, 예시, 단계, 조건을 명확히 합니다.
- CoT는 단계별 사고를 유도합니다.
- Prompt Injection은 기존 지시 무시/탈취, Jailbreaking은 금지된 작업 우회입니다.
- AI Safety 유형과 네이버 AI 윤리 원칙을 구분해서 기억합니다.

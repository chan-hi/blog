## 핵심 요약

- Cloud Functions: 액션 최대 48MB, Node.js/Python/Java/Swift/PHP/.NET/Go 지원
- SourceCommit: Private Git, FileSafer 연동
- SourceBuild: Ubuntu 16.04 기반, GitHub 소스도 지원
- SourceDeploy: 에이전트 필수 설치, Rolling/Blue-Green/Canary 배포
- SourcePipeline: SourceCommit + SourceBuild + SourceDeploy 세 서비스 모두 필요
- SourceBand: Free(프로젝트 3개) / Basic(무제한, 5명 초과 시 5,000원/월)

## Cloud Functions (서버리스)

이벤트 기반 서버리스 컴퓨팅 플랫폼.

### 특징

- **액션(Action)**: 함수 실행 단위, 최대 **48MB**
- 트리거: HTTP, Messaging, Cloud DB 이벤트 등
- 실행 시간 과금 (유휴 시 비용 없음)

### 지원 런타임

| 언어 | 지원 |
|------|------|
| Node.js | ✅ |
| Python | ✅ |
| Java | ✅ |
| Swift | ✅ |
| PHP | ✅ |
| .NET | ✅ |
| Go | ✅ |

### 구성 요소

| 요소 | 설명 |
|------|------|
| **Action** | 단일 함수 실행 단위 |
| **Trigger** | 이벤트 소스 (HTTP/메시지 등) |
| **Rule** | Trigger → Action 연결 규칙 |
| **Package** | Action/Feed의 그룹 |
| **Sequence** | Action을 순서대로 연결 실행 |

## Source 시리즈 CI/CD

### SourceCommit (형상 관리)

- **Private Git Repository** 서비스
- NCP 내 개인화된 코드 저장소
- **FileSafer 연동** → 악성코드 스캔
- CodeReview 기능 제공
- GitHub/Bitbucket 미러링 지원

### SourceBuild (빌드)

- 소스코드 **빌드 자동화** 서비스
- 빌드 환경: **Ubuntu 16.04** 기반
- **GitHub 소스도 지원** (SourceCommit 외부 소스)
- 빌드 결과물: Object Storage 저장

### SourceDeploy (배포)

- **자동 배포** 서비스
- **에이전트 필수 설치** (배포 대상 서버에 Agent 설치)
- 배포 대상: 서버, Auto Scaling Group, Kubernetes

#### 배포 전략 비교

| 전략 | 설명 | 특징 |
|------|------|------|
| **Rolling** | 기존 인스턴스를 순차적으로 교체 | 무중단, 배포 중 구버전/신버전 혼재 |
| **Blue-Green** | 동일 환경 2세트 → 트래픽 전환 | 빠른 롤백, 비용 2배 |
| **Canary** | 일부 트래픽만 신버전으로 전환 | 위험 최소화, 점진적 배포 |

### SourcePipeline (파이프라인)

- 빌드~배포 **통합 파이프라인** 구성
- **세 서비스 모두 필요**: SourceCommit + SourceBuild + SourceDeploy
- 스테이지 구성: 소스 → 빌드 → 배포

### SourceBand (프로젝트 관리)

- 소프트웨어 개발 **올인원 협업 플랫폼**
- 이슈 트래킹, Wiki, CI/CD 통합

| 플랜 | 프로젝트 | 인원 | 가격 |
|------|---------|------|------|
| **Free** | **3개** | 5명 | 무료 |
| **Basic** | **무제한** | 5명 초과 시 | **5,000원/월/인** |

## 클라우드 아키텍처 원칙

| 원칙 | 설명 |
|------|------|
| **Availability** | 가용성 — 장애 없이 지속 제공 |
| **Scalability** | 확장성 — 부하에 따라 증설 |
| **Elasticity** | 탄력성 — 필요에 따라 자동 증감 |
| **Automated** | 자동화 — 수동 개입 최소화 |
| **Security** | 보안 — 보호, 규정 준수 |

## 모놀리식 vs MSA

### 아키텍처 진화

```
Monolithic → SOA → Microservice
```

| 항목 | Monolithic | SOA | Microservice |
|------|-----------|-----|-------------|
| **구조** | 단일 애플리케이션 | 서비스 모듈화 | 독립 마이크로서비스 |
| **배포** | 전체 재배포 | ESB 통해 연동 | 각 서비스 독립 배포 |
| **장점** | 단순, 개발 빠름 | 재사용성 향상 | 독립 배포, 기술 다양성 |
| **단점** | 스케일 어려움 | ESB 병목 가능 | 복잡한 운영 |

### 클라우드 마이그레이션 전략

| 전략 | 설명 |
|------|------|
| **Lift-and-Shift** | 기존 시스템 그대로 클라우드 이전 (Rehost) |
| **Cloud-optimized** | 클라우드 기능 일부 활용하여 최적화 (Replatform) |
| **Cloud-native** | 클라우드 기반으로 완전히 재설계 (Rearchitect) |

## DevOps / GitOps

| 개념 | 설명 |
|------|------|
| **DevOps** | 개발(Dev) + 운영(Ops) 통합 문화 및 자동화 |
| **GitOps** | Git을 단일 진실 원본으로 인프라/배포 관리 |
| **IaC** | 인프라를 코드로 정의 (Terraform, Ansible 등) |
| **Blue-Green** | 무중단 배포 전략, 트래픽 즉시 전환 |
